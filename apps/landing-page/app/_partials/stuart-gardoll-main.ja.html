背景

彼はあらゆるモデルを試す。必要だったのは、ひとつのクリエイティブの土台だった。

Stuart は一人スタジオ Connect I/O を営み、各社の AI 搭載アプリのリリースを支援している — そして AI でものづくりをする YouTube チャンネル Let’s Build を運営している。(その前は FlutterFlow の Growth Engineer だった。) 最新であり続けることが仕事そのものだから、新しいツール、モデル、コーディングエージェントの間を絶えず行き来している。彼が求めているのは、完璧なモデルが 1 つあることではない — 下にあるモデルが変わり続けても、そこに留まり続けるクリエイティブの土台だ。

彼はそれまで Claude Design を使っていて、その方向性を気に入っていた。だがそれは彼を 1 社のモデルに縛りつけた。週単位でモデルを切り替える人間にとって、そのロックインは形が合わなかった。そんなとき、知人がオープンソースの代替ツールについて投稿した — “I always like to support the open-source community wherever I can, so I tried it out.”

It does everything Claude Design can do — but with the model that I choose. And it goes even a bit further, because it’s not so hamstrung.
Stuart Gardoll · Connect I/O
実証 · アプリ制作のワークフロー

リリース済みのアプリが、Open Design で形になった。

Stuart の Read It — どんな記事でも音声に変えるアプリ — は App Store と Google Play で公開中だ。そのインターフェースが形になる場所が Open Design だった。読み上げの流れ、ローディングの演出、人間味を感じさせる細やかなタイミングの一つひとつ。彼はそこで方向性をデザインし、成果物をコーディングエージェントに引き渡して、自分の Flutter と GCP のスタックで作り上げていく。

“I create whatever I need, then I hand it off to my preferred CLI. It flows really nicely.”

実証 · コンテンツ制作のワークフロー

彼の動画に登場するモーショングラフィックス — その制作は Open Design。

もう一つのワークフローがメディアだ。Stuart の Let’s Build 動画の一部は、そのモーショングラフィックスを Hyperframes を使って Open Design で作っている。自分のブランドカラーで欲しいものを言葉で伝えると、Open Design がそれを生成し、その場でライブプレビューし、MP4 に書き出す — そのまま編集に放り込める、視聴者が期待するプロダクション品質で。

“My latest video uses Open Design for all of its motion graphics.”

そして、それだけにとどまらない

何を作るときも、ひとつの土台で。

プロトタイプ — リポジトリから

既存の GitHub リポジトリを取り込み、Open Design で作り直す — 荒削りなツールが、そのままリリースできる洗練されたものに変わる。

動くアーティファクト

ある朝の実験で、Hacker News を Flipboard 風の新聞に仕立てた — “something that’s been amazing”、それをそのままソフトウェアに落とし込む。

自分だけのデザインシステム

プリセットではなく — 自分だけのデザインシステムや、繰り返し使えるビジュアルの方向性を試し、それをもとに生成していく。

なぜ彼の定番なのか

すでに使っているすべてに、そのままつながる。

Open Design は Stuart のスタックの残りの部分にすっと収まる。Figma からインポートし、CLI やコーディングエージェントへきれいに受け渡し、さらには日々のプロジェクトツールにも組み込もうと考えている。“I like that the platform is so connected to everything else.” あらゆるツールを試す人間が、あちこち探し回るのをやめた理由がこれだ。
