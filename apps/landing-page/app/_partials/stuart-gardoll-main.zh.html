背景

他试遍每一个模型,要的只是一块统一的创作画布。

Stuart 经营着一家一个人的工作室 Connect I/O,帮企业交付 AI 驱动的应用 — 同时他还运营着 Let’s Build,一个讲如何用 AI 做产品的 YouTube 频道。(在那之前,他是 FlutterFlow 的 Growth Engineer。)紧跟前沿本身就是工作,所以他不停地在新工具、新模型和各种 coding agent 之间切换。他要的不是某一个完美的模型 — 而是一块底层模型不断更替、它却始终不变的创作画布。

他一直在用 Claude Design,也认可它的路子。但它把他绑死在一家公司的模型上,而对一个每周都在换模型的人来说,这种锁定完全不对味。后来,一位熟人发帖介绍了一个开源的替代品 — “I always like to support the open-source community wherever I can, so I tried it out.”

It does everything Claude Design can do — but with the model that I choose. And it goes even a bit further, because it’s not so hamstrung.
Stuart Gardoll · Connect I/O
实证 · 一条应用工作流

一个已上线的应用,在 Open Design 里成形。

Stuart 的 Read It — 一个能把任何文章转成音频的应用 — 已上线 App Store 和 Google Play。界面正是在 Open Design 里成形的:阅读流程、加载时的动效编排,还有那些让产品有了人情味、精心掐好时机的小瞬间。他在里面把方向定下来,再把产物交给自己的 coding agent,在他的 Flutter 和 GCP 技术栈里落地实现。

“I create whatever I need, then I hand it off to my preferred CLI. It flows really nicely.”

实证 · 一条内容工作流

他视频里的动态图形 — 都是在 Open Design 里做的。

另一条工作流是媒体内容。Stuart 的一些 Let’s Build 视频,动态图形就是用 Hyperframes 在 Open Design 里做的:他用自己的品牌配色描述想要的效果,Open Design 便生成出来、实时预览,再导出成 MP4 — 直接拖进剪辑就能用,而且是他的观众所期待的那种成片级质量。

“My latest video uses Open Design for all of its motion graphics.”

还远不止这些

无论他在做什么,都是同一块画布。

原型 — 从一个 repo 开始

他会拿一个现成的 GitHub repo,在 Open Design 里重新构想一遍 — 一个粗糙的工具,变成真正能交付的干净成品。

能跑起来的产物

一个上午的小实验,把 Hacker News 变成一份 Flipboard 风格的报纸 — “something that’s been amazing”,然后他直接把它放进了软件里。

他自己的设计系统

不是用预设 — 他会拿自己的设计系统和可复用的视觉方向来做实验,再照着它们生成。

为什么它成了他的默认之选

它能接进他早已在用的一切。

Open Design 严丝合缝地嵌进了 Stuart 技术栈的其余部分:他从 Figma 导入,干净利落地交接给自己的 CLI 和 coding agent,还很想把它接进日常的项目工具里。“I like that the platform is so connected to everything else.” 对一个逢新工具必试的人来说,这正是他不再到处比较挑选的原因。
