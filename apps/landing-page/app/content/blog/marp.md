      <p>如果你搜索"marp"是想找一个牙科或正畸术语，那这是另一个：<strong>Marp 是 Markdown 演示文稿生态（Markdown Presentation Ecosystem）</strong>——一个来自 GitHub 上 <code>marp-team</code> 的开源（MIT）项目，通过编写纯 Markdown 来构建幻灯片。不用在画布上拖拽方框，也没有私有文件格式。你写下 <code># Slide title</code> 和几个要点，加上一两条指令，Marp 就把它变成一份演示文稿。（你可能见过的、与此无关的"MARPE"是正畸学里的一种上颌扩弓手术——跟幻灯片毫无关系。本文只讲 Marp 这个演示文稿工具。）</p>
      <p>在开源演示文稿工具里，Marp 是最简单、最贴近 Markdown 原生的选择，这正是开发者青睐它的原因。这是一份诚实的指南，讲清楚 Marp 是什么、编写与主题如何运作、Marp CLI 如何导出到 PowerPoint 和 PDF（以及那个总让人踩坑的注意事项）、它与 reveal.js 和 Slidev 的对比，以及当 Markdown 走到尽头时，设计工作空间该如何接手。</p>
      <h2>Marp 是什么</h2>
      <p>Marp 是围绕一个理念构建的一小套工具：<strong>幻灯片即 Markdown</strong>。它由三个部分组成。<strong>Marpit</strong> 是底层框架，把 Markdown 加上主题转换成幻灯片的 HTML 和 CSS。<strong>Marp Core</strong> 是构建在 Marpit 之上、开箱即用的引擎，自带默认主题、图片处理和数学公式支持。<strong>Marp CLI</strong>（<code>marp-team/marp-cli</code>）是命令行工具，把你的 Markdown 文件渲染成 HTML、PDF 或 PowerPoint。还有一个官方的 <strong>VS Code 扩展</strong>，能在你输入时提供实时的并排预览。</p>
      <p>你在一个 <code>.md</code> 文件里编写。幻灯片之间用 <code>---</code> 分隔，逐页和全局设置通过 Marp 指令（如 <code>theme:</code>、<code>paginate:</code> 或 <code>class:</code>）来设定，一切都保持为可版本控制的纯文本。这正是它的全部魅力：你的幻灯片在 Git 里能干净地 diff、跟你的代码放在一起，也永远不会把你锁进某种二进制格式。</p>
      <h2>Marp 主题与 Markdown 编写</h2>
      <p>编写一份 Marp 幻灯片刻意保持极简。文件顶部的 front-matter 块设定全局指令，<code>---</code> 分隔幻灯片，而标准的 Markdown——标题、列表、代码块、图片、表格——就成了你的内容。因为 Marp 底层渲染成 HTML 和 CSS，任何 Markdown 能表达的东西都能映射到一张幻灯片，无需再学额外的语法。</p>
      <p><strong>Marp 主题</strong>就是 CSS。Marp Core 自带几个内置主题（<code>default</code>、<code>gaia</code>、<code>uncover</code>），你用一条 <code>theme:</code> 指令就能在它们之间切换。当内置主题不够用时，你把自己的主题写成一个 CSS 文件——针对 Marpit 暴露的类，通过 <code>--theme</code> 或 <code>--theme-set</code> 用 CLI 注册它，每一张幻灯片都会套用它。想做行内微调，你可以往某一张幻灯片里放一个带作用域的 <code>&lt;style&gt;</code> 块。这种 CSS 原生的模型既是 Marp 的天花板也是地板：如果你熟悉 CSS，就拥有完全的掌控力；如果不熟，默认样式看起来也很干净。</p>
      <h2>把 Marp 导出到 PowerPoint 和 PDF</h2>
      <p>导出正是 Marp CLI 体现价值的地方。把它指向你的 Markdown，选一个目标格式：</p>
      <ul><li><strong>HTML</strong>——<code>marp deck.md -o deck.html</code> 会生成一份自包含的网页幻灯片，供你在浏览器里演示。</li><li><strong>PDF</strong>——<code>marp deck.md --pdf</code> 渲染出一份可打印的 PDF，非常适合作讲义和分享。</li><li><strong>PowerPoint</strong>——<code>marp deck.md --pptx</code> 导出一个 <code>.pptx</code> 文件，你可以在 PowerPoint 或 Keynote 里打开。</li></ul>
      <p>有一个值得直说的注意事项：<strong>Marp 的 PPTX 导出会把每一张幻灯片渲染成一张图片</strong>，而不是原生、可编辑的 PowerPoint 形状。每张幻灯片都变成 <code>.pptx</code> 容器里一张满版的图。当你需要把文件交给一个只需演示、离不开 PowerPoint 的人时，这堪称完美——但导出之后，你没法点进去编辑文本框、给形状换色，或挪动某个要点。如果你的工作流之后还要在 PowerPoint 里继续编辑，就把 Marp 的 pptx 当成交付格式，而不是源格式。（实验性的"可编辑"导出模式来了又走；可靠且有文档记载的行为是一页一图。）</p>

      <h2>Open Design 的位置</h2>
      <p>Marp 在一件事上做得极好：把你已经写好的 Markdown 变成幻灯片。但它要求<em>你</em>来写这份幻灯片——结构、措辞、主题 CSS。它不会生成内容、不会让内容跨项目符合品牌调性，也不会在导出后给你一份可编辑的成果。</p>
      <p><a href="/">Open Design</a> 是它之上的那一层。它是一个开源（Apache-2.0）、本地优先、自备 key 的 <strong>Agent-Native Design Workspace</strong>，位于你已经在用的编程 Agent 之外。你不必手写 Markdown，而是描述你想要的幻灯片，Agent 就对着一套设计系统生成一份可编辑的——它的<a href="/plugins/templates/">插件库</a>里还提供了第一方的 HTML 幻灯片模板作为起点。它并不是要取代用 Markdown 编写的 Marp：当你已经把内容写进 Markdown、想从命令行快速得到一份幻灯片时，Marp 就是对的工具。当你想让幻灯片被生成出来、符合品牌调性，并且作为一整套更大的设计工作流的一部分仍然可编辑时，那就是工作空间的活了。如果你已经在用编程 Agent 做幻灯片，看看<a href="/blog/claude-ppt-skills/">编程 Agent 如何做幻灯片</a>。</p>
      <h2>常见问题</h2>
      <p><strong>什么是 Marp？</strong>Marp 是来自 GitHub 上 <code>marp-team</code> 的开源 Markdown 演示文稿生态。你用 Markdown 编写幻灯片，Marp CLI 把它们导出成 HTML、PDF 或 PowerPoint。它采用 MIT 许可证，包含 Marp Core、Marpit 框架和一个 VS Code 扩展。它与正畸学里的"MARPE"一词无关。</p>
      <p><strong>我怎么把一份 Marp 幻灯片导出到 PowerPoint？</strong>安装 Marp CLI（<code>marp-cli</code>）并运行 <code>marp deck.md --pptx</code>。注意导出的 <code>.pptx</code> 会把每张幻灯片渲染成图片，而不是原生可编辑的 PowerPoint 形状——它很适合交接，但导出之后你没法在 PowerPoint 里编辑文本或形状。</p>
      <p><strong>Marp 能导出可编辑的 PowerPoint 文件吗？</strong>不太可靠。Marp 有文档记载的 PPTX 行为是一页一图。如果你需要带真实文本框和形状、原生可编辑的 <code>.pptx</code>，那这一步 Marp 就是错的工具。</p>
      <p><strong>Marp 免费吗？</strong>免费。Marp 在 MIT 许可证下开源，因此可以自由使用、自托管和修改。</p>
      <p><strong>Marp、reveal.js 与 Slidev——我该用哪个？</strong>要最简单的纯 Markdown 幻灯片以及便捷的 PDF/pptx 导出，选 Marp；要一个成熟、以 HTML 为先、可深度定制的网页幻灯片框架，选 reveal.js；要 Markdown 加上交互式 Vue 组件，选 Slidev。这三者都采用 MIT 许可证，都无法直接导出可编辑的 PowerPoint。</p>
      <h2>结语</h2>

      <h2>Marp とは</h2>
      <p>Marp は一つの発想を中心に構築された小さなツール群です：<strong>スライドは Markdown</strong>。三つの構成要素があります。<strong>Marpit</strong> は、Markdown とテーマをスライドの HTML と CSS に変換する低レベルのフレームワークです。<strong>Marp Core</strong> は Marpit の上に載る、デフォルトテーマ・画像処理・数式サポートを備えた「電池同梱」のエンジンです。<strong>Marp CLI</strong>（<code>marp-team/marp-cli</code>）は、あなたの Markdown ファイルを HTML、PDF、PowerPoint にレンダリングするコマンドラインツールです。さらに、入力しながらリアルタイムで横並びプレビューを見せてくれる公式の <strong>VS Code 拡張</strong>もあります。</p>
      <p>執筆は一つの <code>.md</code> ファイルで行います。スライドは <code>---</code> で区切り、スライドごとおよびグローバルの設定は Marp のディレクティブ（<code>theme:</code>、<code>paginate:</code>、<code>class:</code> など）で指定し、すべてがバージョン管理可能なプレーンテキストのままです。それこそが最大の魅力です。あなたのデッキは Git できれいに差分が取れ、コードの隣に置け、バイナリ形式に縛られることもありません。</p>
      <h2>Marp のテーマと Markdown での執筆</h2>
      <p>Marp デッキの執筆は意図的に最小限です。ファイル冒頭の front-matter ブロックがグローバルディレクティブを設定し、<code>---</code> がスライドを区切り、標準の Markdown——見出し、リスト、コードフェンス、画像、表——がそのままコンテンツになります。Marp は内部で HTML と CSS にレンダリングするため、Markdown で表現できるものは追加の構文を覚えることなくスライドに対応します。</p>
      <p><strong>Marp のテーマ</strong>はただの CSS です。Marp Core はいくつかの組み込みテーマ（<code>default</code>、<code>gaia</code>、<code>uncover</code>）を同梱しており、たった一つの <code>theme:</code> ディレクティブで切り替えられます。組み込みでは足りないときは、自分のテーマを CSS ファイルとして書きます——Marpit が公開するクラスを対象にし、<code>--theme</code> または <code>--theme-set</code> で CLI に登録すれば、すべてのスライドがそれを取り込みます。インラインの微調整には、一枚のスライドにスコープ付きの <code>&lt;style&gt;</code> ブロックを差し込めます。この CSS ネイティブなモデルこそが Marp の天井であり床でもあります。CSS に慣れていれば完全に制御でき、そうでなくてもデフォルトのままきれいに見えます。</p>
      <h2>Marp を PowerPoint と PDF に書き出す</h2>
      <p>書き出しこそ Marp CLI が真価を発揮する場面です。Markdown を指定し、ターゲットを選びます：</p>
      <ul><li><strong>HTML</strong>——<code>marp deck.md -o deck.html</code> は、ブラウザで発表できる自己完結型の Web デッキを生成します。</li><li><strong>PDF</strong>——<code>marp deck.md --pdf</code> は印刷対応の PDF をレンダリングし、配布資料や共有に最適です。</li><li><strong>PowerPoint</strong>——<code>marp deck.md --pptx</code> は、PowerPoint や Keynote で開ける <code>.pptx</code> ファイルを書き出します。</li></ul>
      <p>はっきり述べておくべき一つの注意点があります：<strong>Marp の PPTX 書き出しは各スライドを画像としてレンダリングします</strong>。ネイティブで編集可能な PowerPoint の図形ではありません。すべてのスライドは <code>.pptx</code> ラッパーの中で全面の画像になります。PowerPoint から離れられない、発表するだけの相手にファイルを渡すときには完璧です——しかし書き出した後は、テキストボックスに入って編集したり、図形の色を変えたり、箇条書きを動かしたりはできません。あとで PowerPoint で編集し続ける必要があるワークフローなら、Marp の pptx はソース形式ではなく配布形式として扱いましょう。（実験的な「編集可能」書き出しモードは現れては消えてきました。信頼でき、文書化された挙動は一スライド一画像です。）</p>

      <h2>Open Design が活きる場面</h2>
      <p>Marp はまさに一つのことに秀でています：あなたがすでに書いた Markdown をスライドに変えることです。しかしそれは<em>あなた</em>にデッキを書くこと——構成、言い回し、テーマ CSS——を求めます。コンテンツを生成したり、プロジェクトをまたいでブランドに沿って保ったり、書き出した後に編集可能な成果物を渡したりはしません。</p>
      <p><a href="/">Open Design</a> はその一段上のレイヤーです。すでに使っているコーディングエージェントの外側に位置する、オープンソース（Apache-2.0）、ローカルファースト、キー持ち込み方式の <strong>Agent-Native Design Workspace</strong> です。Markdown を手書きする代わりに、欲しいデッキを説明すると、エージェントがデザインシステムに沿って編集可能なものを生成します——出発点となるファーストパーティの HTML デッキテンプレートを<a href="/plugins/templates/">プラグインライブラリ</a>に備えています。これは Markdown で書く Marp の置き換えではありません。すでに Markdown にコンテンツがあり、コマンドラインから手早くデッキがほしいなら、Marp が適切なツールです。デッキを生成させ、ブランドに沿わせ、より大きなデザインワークフローの一部として編集可能なままにしたいなら、それはワークスペースの仕事です。すでにコーディングエージェントでスライドを作っているなら、<a href="/blog/claude-ppt-skills/">コーディングエージェントでのスライド作成方法</a>をご覧ください。</p>
      <h2>FAQ</h2>
      <p><strong>Marp とは？</strong> Marp は GitHub の <code>marp-team</code> によるオープンソースの Markdown プレゼンテーションエコシステムです。Markdown でスライドを書くと、Marp CLI がそれを HTML、PDF、PowerPoint に書き出します。MIT ライセンスで、Marp Core、Marpit フレームワーク、VS Code 拡張を含みます。矯正歯科の「MARPE」という用語とは関係ありません。</p>
      <p><strong>Marp デッキを PowerPoint に書き出すには？</strong> Marp CLI（<code>marp-cli</code>）をインストールし、<code>marp deck.md --pptx</code> を実行します。書き出された <code>.pptx</code> は各スライドを画像としてレンダリングし、ネイティブで編集可能な PowerPoint の図形ではない点に注意してください——受け渡しには最適ですが、その後 PowerPoint 内でテキストや図形を編集することはできません。</p>
      <p><strong>Marp は編集可能な PowerPoint ファイルを書き出せますか？</strong> 確実にはできません。Marp の文書化された PPTX の挙動は一スライド一画像です。本物のテキストボックスや図形を持つネイティブに編集可能な <code>.pptx</code> が必要なら、その工程には Marp は不向きです。</p>
      <p><strong>Marp は無料ですか？</strong> はい。Marp は MIT ライセンスのオープンソースなので、無料で使え、セルフホストでき、改変もできます。</p>
      <p><strong>Marp、reveal.js、Slidev——どれを使うべき？</strong> 最もシンプルな純 Markdown デッキと手軽な PDF/pptx 書き出しなら Marp、深いカスタマイズができる成熟した HTML ファーストの Web スライドフレームワークなら reveal.js、Markdown に加えてインタラクティブな Vue コンポーネントなら Slidev を選びましょう。三つとも MIT ライセンスで、いずれも編集可能な PowerPoint を直接は書き出しません。</p>
      <h2>まとめ</h2>
      <p>Marp は、すでに手元にある Markdown をスライドに変える最もすっきりした方法です——一つの <code>.md</code> ファイル、CSS テーマ、そして HTML、PDF、PowerPoint を書き出す CLI。ただ一つの正直な制約だけは忘れないでください：pptx は画像として出てくるので、PowerPoint で編集し続けるものではなく配布形式として扱いましょう。「Markdown はある、デッキがほしい」というニーズなら、Marp を超えるものはなかなかありません。デッキを生成させ、ブランドに沿わせ、より大きなデザインワークフローの中で編集可能なままにする必要があるとき——そこで <a href="/">Agent-Native Design Workspace</a> が引き継ぎます。</p>

      <h2>Open Design이 어울리는 지점</h2>
      <p>Marp는 딱 한 가지에 탁월합니다: 당신이 이미 작성한 Markdown을 슬라이드로 바꾸는 것. 하지만 그것은 <em>당신</em>에게 덱을 쓰라고 요구합니다——구조, 문구, 테마 CSS. 콘텐츠를 생성하거나, 프로젝트 전반에 걸쳐 브랜드에 맞게 유지하거나, 내보낸 뒤 편집 가능한 산출물을 주지는 않습니다.</p>
      <p><a href="/">Open Design</a>은 그 위의 레이어입니다. 이미 사용 중인 코딩 에이전트 바깥에 자리한, 오픈소스(Apache-2.0)이고 로컬 우선이며 키 지참 방식의 <strong>Agent-Native Design Workspace</strong>입니다. Markdown을 손으로 쓰는 대신 원하는 덱을 설명하면, 에이전트가 디자인 시스템에 맞춰 편집 가능한 덱을 생성합니다——출발점이 되는 퍼스트파티 HTML 덱 템플릿을 <a href="/plugins/templates/">플러그인 라이브러리</a>에 갖추고 있습니다. 이는 Markdown으로 작성하는 Marp의 대체품이 아닙니다. 이미 Markdown에 콘텐츠가 있고 명령줄에서 빠른 덱을 원한다면 Marp가 알맞은 도구입니다. 덱이 생성되고, 브랜드에 맞고, 더 큰 디자인 워크플로의 일부로 여전히 편집 가능하길 원한다면, 그건 워크스페이스의 몫입니다. 이미 코딩 에이전트로 슬라이드를 만든다면 <a href="/blog/claude-ppt-skills/">코딩 에이전트로 슬라이드 덱 만드는 법</a>을 참고하세요.</p>
      <h2>자주 묻는 질문</h2>
      <p><strong>Marp란 무엇인가요?</strong> Marp는 GitHub의 <code>marp-team</code>이 만든 오픈소스 Markdown 프레젠테이션 생태계입니다. Markdown으로 슬라이드를 작성하면 Marp CLI가 그것을 HTML, PDF, PowerPoint로 내보냅니다. MIT 라이선스이며 Marp Core, Marpit 프레임워크, VS Code 확장을 포함합니다. 교정치과의 "MARPE" 용어와는 무관합니다.</p>
      <p><strong>Marp 덱을 PowerPoint로 어떻게 내보내나요?</strong> Marp CLI(<code>marp-cli</code>)를 설치하고 <code>marp deck.md --pptx</code>를 실행하세요. 내보낸 <code>.pptx</code>는 각 슬라이드를 이미지로 렌더링하며, 네이티브하고 편집 가능한 PowerPoint 도형이 아니라는 점에 유의하세요——전달에는 훌륭하지만 이후 PowerPoint 안에서 텍스트나 도형을 편집할 수는 없습니다.</p>
      <p><strong>Marp는 편집 가능한 PowerPoint 파일을 내보낼 수 있나요?</strong> 확실하게는 아닙니다. Marp의 문서화된 PPTX 동작은 슬라이드당 이미지 하나입니다. 진짜 텍스트 상자와 도형을 갖춘 네이티브 편집 가능한 <code>.pptx</code>가 필요하다면, 그 단계에는 Marp가 맞지 않는 도구입니다.</p>
      <p><strong>Marp는 무료인가요?</strong> 네. Marp는 MIT 라이선스 오픈소스이므로 자유롭게 사용하고, 셀프호스팅하고, 수정할 수 있습니다.</p>

No dragging boxes on a canvas, no proprietary file format. You write `# Slide title` and a few bullets, add a directive or two, and Marp turns it into a presentation. (The unrelated "MARPE" you may have seen is a maxillary expansion procedure in orthodontics — nothing to do with slides. This article is strictly about Marp, the presentation tool.)

Marp is the simplest, most Markdown-native option among the open-source presentation tools, which is exactly why developers reach for it. This is an honest guide to what Marp is, how authoring and themes work, how the Marp CLI exports to PowerPoint and PDF (and the one caveat that catches people out), how it stacks up against reveal.js and Slidev, and where a design workspace picks up when Markdown runs out of road.

## What Marp is

Marp is a small ecosystem of tools built around one idea: **slides as Markdown**. It has three moving parts. **Marpit** is the low-level framework that turns Markdown plus a theme into slide HTML and CSS. **Marp Core** is the batteries-included engine on top of Marpit, with a default theme, image handling, and math support. **Marp CLI** (`marp-team/marp-cli`) is the command-line tool that renders your Markdown file into HTML, PDF, or PowerPoint. There's also an official **VS Code extension** that gives you a live side-by-side preview as you type.

You author in one `.md` file. Slides are separated by `---`, per-slide and global settings are set through Marp directives (like `theme:`, `paginate:`, or `class:`), and everything stays in version-controllable plain text. That's the whole appeal: your deck diffs cleanly in Git, lives next to your code, and never locks you into a binary format.

## Marp themes and Markdown authoring

Authoring a Marp deck is deliberately minimal. A front-matter block at the top of the file sets global directives, `---` breaks slides, and standard Markdown — headings, lists, code fences, images, tables — becomes your content. Because Marp renders to HTML and CSS under the hood, anything Markdown expresses maps to a slide without extra syntax to learn.

**Marp themes** are just CSS. Marp Core ships a handful of built-in themes (`default`, `gaia`, `uncover`), and you switch between them with a single `theme:` directive. When the built-ins aren't enough, you write your own theme as a CSS file — target the classes Marpit exposes, register it with the CLI via `--theme` or `--theme-set`, and every slide picks it up. For inline tweaks you can drop a scoped `<style>` block into a single slide. This CSS-native model is Marp's real ceiling and floor: if you're comfortable with CSS you have full control, and if you're not, the defaults still look clean.

## Exporting Marp to PowerPoint and PDF

Export is where the Marp CLI earns its place. Point it at your Markdown and choose a target:

- **HTML** — `marp deck.md -o deck.html` produces a self-contained web deck you present in the browser.
- **PDF** — `marp deck.md --pdf` renders a print-ready PDF, ideal for handouts and sharing.
- **PowerPoint** — `marp deck.md --pptx` exports a `.pptx` file you can open in PowerPoint or Keynote.

The one caveat worth stating plainly: **Marp's PPTX export renders each slide as an image**, not as native, editable PowerPoint shapes. Every slide becomes a full-bleed picture inside the `.pptx` wrapper. That's perfect when you need to hand a file to someone who lives in PowerPoint and only has to present it — but you can't click in and edit a text box, recolor a shape, or nudge a bullet once it's exported. If your workflow needs to keep editing in PowerPoint afterward, treat Marp's pptx as a delivery format, not a source format. (Experimental "editable" export modes have come and gone; the reliable, documented behavior is image-per-slide.)

## Marp vs Slidev vs reveal.js

**Marp** is the choice when you want the least ceremony — a single Markdown file and a CLI. **[reveal.js](/blog/reveal-js/)** is the mature web-slides framework: HTML-first, endlessly customizable, and the richest in-browser experience, but you're writing HTML and there's no native PowerPoint path. **[Slidev](/blog/slidev/)** sits in the middle for developers who want Markdown plus real Vue components, code highlighting, and interactive embeds — more power than Marp, more setup too. All three are MIT-licensed and export nothing directly editable in PowerPoint.

## Where Open Design fits

Marp is excellent at exactly one thing: taking Markdown you've already written and turning it into slides. But it asks *you* to write the deck — the structure, the wording, the theme CSS. It doesn't generate the content, keep it on-brand across projects, or give you an editable artifact once it's exported.
