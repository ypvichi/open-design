      <h2>什么是 Claude PPT 技能？</h2>
      <p>技能是一套打包好的指令和素材，编程 Agent（Claude Code、Codex、Cursor 等）会在任务进行中加载它，去完成某件具体的事。一个 <strong>Claude PPT 技能</strong>教会 Agent 生成演示文稿：你用大白话描述这套幻灯片，Agent 套用技能里的版式系统和设计规则，然后把幻灯片交还给你——通常是 HTML，或者一份可编辑的 PowerPoint 文件。</p>
      <p>它回答的正是人们真正会去搜索的问题：</p>
      <ul><li><strong>Claude 能做 PPT 吗？</strong>能——装一个 PPT 技能，然后给它提示词。</li><li><strong>我怎么用 Claude 做 PPT？</strong>描述主题、受众和篇幅；技能会处理结构、版式和样式。</li><li><strong>它免费吗？</strong>这些技能大多是开源且免费的；你只需自备模型的 key。</li></ul>
      <p>相比一个完整的应用，它的取舍在于：你是通过 Agent 和一个代码仓库来工作，而不是一个打磨精致的图形界面——而这恰恰是开发者喜欢它们的原因。</p>

      <h2>如何使用一个 Claude PPT 技能</h2>
      <p>各个技能的流程大同小异：</p>
      <ol><li><strong>安装技能</strong>——把技能克隆或添加到你的编程 Agent 的技能目录里（每个仓库都有配置步骤）。自备你的模型 key。</li><li><strong>描述这套幻灯片</strong>——给 Agent 提示词，说明主题、受众、语气和大致页数。简报越具体，结构越好。</li><li><strong>让技能运行</strong>——Agent 套用技能的版式系统并生成幻灯片。</li><li><strong>迭代</strong>——用大白话要求修改（"把开头收紧一点"、"把第 4 页做成对比表"、"换成编辑排版主题"）然后重新渲染。</li><li><strong>导出</strong>——导成 HTML、PDF，或（对支持的技能而言）一份可编辑的 <code>.pptx</code>。</li></ol>
      <h2>从技能到工作空间：Open Design 的位置</h2>
      <p>一个 PPT 技能，用来在你的终端里做一份一次性的幻灯片再合适不过。但技能本质上是一段脚本——它不会把你的品牌带到不同项目之间，不会让输出在一个真正的工作空间里保持可编辑，也不会让这套幻灯片和你其余的设计工作协同起来。</p>
      <p><a href="/">Open Design</a> 是技能之上的那一层：一个开源（Apache-2.0）、本地优先、自备 key 的 <strong>agent-native 设计工作空间</strong>，它独立于你已经在用的编程 Agent 之外。你描述一套幻灯片，Agent 就对着一套设计系统生成一份可编辑的——而且 Open Design 在它的<a href="/plugins/templates/">插件库</a>里提供了第一方的 HTML 幻灯片模板，包括 guizang 风格的编辑排版版式。</p>
      <ul><li><strong>一致，而非一次性</strong>——幻灯片对着一套设计系统产出，符合品牌调性，而不是每次都从一个新模板开始。</li><li><strong>可编辑、且归你所有的输出</strong>——Apache-2.0 且本地优先，所以工作流和成果都属于你。</li><li><strong>同一个 Agent，更大的施展面</strong>——如果你今天已经在用编程 Agent 做幻灯片，看看 <a href="/agents/claude-code-design/">Open Design 如何与 Claude Code 协作</a>。</li></ul>
      <p>当你想从命令行快速得到一份幻灯片时，用技能。当这套幻灯片必须符合品牌调性、可编辑，并且是一整套更大的设计工作的一部分时，去找一个工作空间。</p>
      <h2>常见问题</h2>
      <p><strong>Claude 能做 PowerPoint 吗？</strong>能。装一个 Claude PPT 技能（比如 frontend-slides 或 dashiAI），把你的主题作为提示词交给它——Agent 就会生成幻灯片。有些技能直接导出可编辑的 <code>.pptx</code>；另一些产出你在浏览器里演示的 HTML。</p>
      <p><strong>有没有官方的 Claude PPT 技能？</strong>如今强势的那些选择都是开源社区技能，而非某个官方的 Anthropic 产品。Anthropic 为 Claude/Claude Code 提供了一套技能系统；上面这些演示技能是社区在它之上构建的。</p>
      <p><strong>Claude PPT 技能是免费的吗？</strong>本指南里的这些都是开源且可免费使用的——你自备模型的 API key。在把某个技能嵌进商业产品之前，先看清楚各个项目的许可证（MIT 还是 AGPL-3.0）。</p>
      <p><strong>我怎么用 Claude 做 PPT？</strong>装一个 PPT 技能，然后向你的 Agent 描述这套幻灯片（主题、受众、篇幅）。用大白话迭代，再导出成 HTML 或 PowerPoint。</p>
      <p><strong>技能还是设计工作空间——我该用哪个？</strong>技能最适合在终端里做一份快速、一次性的幻灯片。像 Open Design 这样的 agent-native 工作空间，则更适合你需要符合品牌调性、可编辑、并且与其余设计工作放在一起的幻灯片。</p>
      <h2>结语</h2>
      <p>用编程 Agent 做幻灯片，在 2026 年从一件新鲜事变成了一套真正的工作流。如果你想要一份快速的幻灯片，装一个 Claude PPT 技能——frontend-slides 做 web 原生幻灯片，guizang 求编辑排版的精致，dashiAI 要可编辑的输出。而当这套幻灯片需要符合品牌调性、可编辑，并且成为某个更大整体的一部分时，那就该由一个 <a href="/">agent-native 设计工作空间</a>来接手了。</p>

      <ol><li><strong>スキルをインストールする</strong>——コーディングエージェントのスキルディレクトリにスキルをクローンまたは追加します（各リポジトリにセットアップ手順があります）。モデルのキーは自分で用意します。</li><li><strong>デッキを説明する</strong>——トピック、対象読者、トーン、おおよそのスライド枚数をエージェントにプロンプトで伝えます。ブリーフが具体的であるほど、構成もよくなります。</li><li><strong>スキルを走らせる</strong>——エージェントがスキルのレイアウトシステムを適用し、スライドを生成します。</li><li><strong>反復する</strong>——平易な言葉で修正を依頼し（「導入を引き締めて」「スライド4を比較表にして」「編集デザインのテーマに切り替えて」）、再レンダリングします。</li><li><strong>エクスポートする</strong>——HTML、PDF、または（対応スキルなら）編集可能な<code>.pptx</code>へ。</li></ol>
      <h2>スキルからワークスペースへ：Open Designが役立つ場面</h2>
      <p>PPTスキルは、ターミナル内で一度きりのデッキを作るには完璧です。しかしスキルは所詮スクリプトです——プロジェクトをまたいでブランドを維持したり、出力を本物のワークスペースで編集可能なまま保ったり、デッキを他のデザイン作業全体と連携させたりはできません。</p>
      <p><a href="/">Open Design</a>は、スキルの一段上のレイヤーです。すでに使っているコーディングエージェントの外側に位置する、オープンソース（Apache-2.0）、ローカルファースト、キー持ち込み方式の<strong>エージェントネイティブなデザインワークスペース</strong>です。あなたがデッキを説明すると、エージェントがデザインシステムに沿って編集可能なデッキを生成します——さらにOpen Designは、guizang風の編集デザインレイアウトを含むファーストパーティのHTMLデッキテンプレートを、<a href="/plugins/templates/">プラグインライブラリ</a>で提供しています。</p>
      <ul><li><strong>一度きりではなく、一貫している</strong>——デッキは毎回新しいテンプレートではなく、デザインシステムに沿ってブランドに調和した状態で仕上がります。</li><li><strong>自分のものになる編集可能な出力</strong>——Apache-2.0でローカルファーストなので、ワークフローも成果物もあなたのものです。</li><li><strong>同じエージェントで、より広い作業面</strong>——今日コーディングエージェントでスライドを作っているなら、<a href="/agents/claude-code-design/">Open DesignがClaude Codeとどう連携するか</a>をご覧ください。</li></ul>
      <p>コマンドラインから手早くデッキがほしいときはスキルを使いましょう。デッキがブランドに調和し、編集可能で、より大きなデザイン作業の一部である必要があるときは、ワークスペースに手を伸ばしてください。</p>

"Can Claude make a PowerPoint?" is one of the most-asked questions about coding agents right now — and the answer is yes, through a **Claude PPT skill**: a small, installable skill that teaches your agent to turn a prompt into a full slide deck. The community shipped a wave of them in 2026, several with tens of thousands of GitHub stars.

This is an honest guide to the ones worth your time. What a Claude PPT skill actually is, the leading open-source options, a scorecard, how to install and use one, and — the part most lists skip — where a skill stops being enough and an agent-native design workspace takes over.

## What is a Claude PPT skill?

A skill is a packaged set of instructions and assets that a coding agent (Claude Code, Codex, Cursor and others) loads mid-task to do something specific. A **Claude PPT skill** teaches the agent to generate a presentation: you describe the deck in plain language, the agent applies the skill's layout system and design rules, and you get slides back — usually as HTML or an editable PowerPoint file.

It answers the questions people actually search:

- **Can Claude make a PPT?** Yes — install a PPT skill and prompt it.
- **How do I make a PPT with Claude?** Describe the topic, audience and length; the skill handles structure, layout and styling.
- **Is it free?** Most of these skills are open source and free; you bring your own model key.

The trade-off versus a full app is that you work through your agent and a repo, not a polished GUI — which is exactly why developers love them.

## The best Claude PPT skills in 2026

These are the open-source, coding-agent PPT skills with real traction. Star counts are from GitHub at time of writing.

### frontend-slides (25k★, MIT)

Zara Zhang's **[frontend-slides](https://github.com/zarazhangrui/frontend-slides)** leans on a coding agent's frontend skills to build slides as web pages — clean, modern, HTML-based decks you present in the browser. It is the most-starred skill in this space and a great default if you want web-native slides and full CSS control. Best for developers comfortable living on the web platform.

### guizang-ppt-skill (20.8k★, AGPL-3.0)

From the Chinese AI creator Guizang, **[guizang-ppt-skill](https://github.com/op7418/guizang-ppt-skill)** generates polished HTML decks in strong editorial styles — magazine layouts and Swiss-grid typography that look designed, not templated. Pick it when you care about the deck reading as a crafted artifact. Note the AGPL-3.0 license if you plan to embed it in a closed product.

### dashiAI-ppt-skill (2.1k★, AGPL-3.0)

**[dashiAI](https://github.com/chuspeeism/dashiAI-ppt-skill)** ("Master PPT") ships a dozen visual themes and a large layout library, and — unusually — exports **browser-editable** presentations. If your deliverable has to end up as something a non-developer can still tweak, dashiAI's editable output is a differentiator. AGPL-3.0.

### codex-ppt-skill (3.4k★, MIT)

**[codex-ppt-skill](https://github.com/ningzimu/codex-ppt-skill)** pairs a coding agent with image generation (GPT-Image-2) to produce image-forward PowerPoint decks — good for visually rich, cover-heavy presentations where each slide is a designed image rather than bullet points. MIT-licensed.

### ppt-master (38k★, MIT)

Not strictly a "skill" but worth knowing: **[ppt-master](https://github.com/hugohe3/ppt-master)** turns any document into a *real, editable* PowerPoint with native shapes and animations, via an agent workflow. If editable `.pptx` with native shapes is the hard requirement, it is the closest open-source option. MIT-licensed.

## How to use a Claude PPT skill

The flow is similar across skills:

1. **Install the skill** — clone or add the skill to your coding agent's skills directory (each repo has setup steps). Bring your own model key.
2. **Describe the deck** — prompt the agent with the topic, audience, tone, and rough slide count. The more specific the brief, the better the structure.
3. **Let the skill run** — the agent applies the skill's layout system and generates the slides.
4. **Iterate** — ask for edits in plain language ("tighten the intro", "make slide 4 a comparison table", "switch to the editorial theme") and re-render.
5. **Export** — to HTML, PDF, or (for skills that support it) an editable `.pptx`.

## From a skill to a workspace: where Open Design fits

A PPT skill is perfect for a one-off deck inside your terminal. But a skill is a script — it does not carry your brand across projects, keep the output editable in a real workspace, or coordinate a deck with the rest of your design work.

[Open Design](/) is the layer above the skill: an open-source (Apache-2.0), local-first, bring-your-own-key **agent-native design workspace** that sits outside the coding agent you already use. You describe a deck; the agent generates an editable one against a design system — and Open Design ships first-party HTML deck templates, including guizang-style editorial layouts, in its [plugin library](/plugins/templates/).

- **Consistent, not one-off** — decks come out on-brand against a design system, not a fresh template each time.
- **Editable output you own** — Apache-2.0 and local-first, so the workflow and the result belong to you.
- **Same agent, more surface** — if you build slides with a coding agent today, see how [Open Design works with Claude Code](/agents/claude-code-design/).

Use a skill when you want a quick deck from the command line. Reach for a workspace when the deck has to be on-brand, editable, and part of a larger body of design work.

## FAQ

**Can Claude make a PowerPoint?** Yes. Install a Claude PPT skill (like frontend-slides or dashiAI) and prompt it with your topic — the agent generates the slides. Some skills export directly to editable `.pptx`; others produce HTML you present in the browser.

**Is there an official Claude PPT skill?** The strong options today are open-source community skills, not an official Anthropic product. Anthropic ships a skills system for Claude/Claude Code; the presentation skills above are built on top of it by the community.

**Are Claude PPT skills free?** The ones in this guide are open source and free to use — you supply your own model API key. Check each project's license (MIT vs AGPL-3.0) before embedding one in a commercial product.

**How do I make a PPT with Claude?** Install a PPT skill, then describe the deck (topic, audience, length) to your agent. Iterate in plain language and export to HTML or PowerPoint.

**Skill or a design workspace — which should I use?** A skill is ideal for a fast, one-off deck in your terminal. An agent-native workspace like Open Design is the better fit when you need on-brand, editable decks that live alongside the rest of your design work.
