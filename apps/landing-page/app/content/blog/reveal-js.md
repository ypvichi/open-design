      <p>如果你写代码，又需要做一场演讲，那你大概率碰到过 <strong>reveal.js</strong>。你看过的开发者大会幻灯片里，有很大一部分背后就是这个开源框架：你用 HTML 或 Markdown 编写每一页幻灯片，它作为一个网页在任意浏览器里运行——内建主题、转场、片段动画、演讲者备注，以及原生的 PDF 导出。凭借 GitHub 上的 71.9k 星标和 MIT 许可证，它就是"不打开 PowerPoint 怎么做幻灯片"这个问题的默认答案。</p>
      <p>这份指南给它画一张实话实说的地图。reveal.js 究竟是什么，主题、模板和示例如何拼到一起，导出 PDF 和 PowerPoint 的真实表现（这里有个坑），它跟其他开源竞争者相比如何，以及——大多数文章都跳过的那部分——这样一个框架在什么时候不再是对的工具、又该由别的东西来接手。如果你正在为一套 <a href="/blog/claude-ppt-skills/">AI 与 Agent 辅助的幻灯片工作流</a>权衡选择，那么弄清楚 reveal.js 是什么、不是什么，会很有帮助。</p>
      <h2>reveal.js 是什么</h2>
      <p>reveal.js 自称"HTML 演示框架"，这个描述很精确。它不是一个你打开后点来点去的应用——它是一个你放进 HTML 文件里的 JavaScript 库。每一页幻灯片都是一个 <code>&lt;section&gt;</code> 元素；把 section 嵌套起来，你就得到了纵向幻灯片。你用 CSS 给一切设定样式，用 data 属性接入转场和片段，然后在浏览器里打开这个页面来演示。因为它的输出<em>本身就是</em>一个网页，凡是 web 能做的——实时代码、内嵌视频、iframe、WebGL、你自己的组件——都能出现在一页幻灯片上。</p>
      <p>取舍就写在定义里：这是一个<strong>面向开发者的框架，而不是图形界面</strong>。你的工具是代码编辑器和浏览器，而不是一块带拖拽手柄的画布。这正是开发者喜欢它的原因——完整的版本控制、完整的 CSS 控制、没有私有文件格式——也正是不会写代码的人会被它劝退的原因。它还是真正免费的：没有账号、没有水印、没有导出上限、没有按席位计费。这份 HTML 归你所有。</p>
      <h2>主题、模板与示例</h2>
      <p>围绕 reveal.js 的大多数搜索，其实都是想在不从零设计的前提下让它<em>看起来</em>好看，所以下面讲讲这几块是怎么拼起来的。</p>
      <p><strong>reveal.js 主题</strong>是内建的起点。框架自带一组现成主题——Black、White、League、Beige、Sky、Night、Serif、Simple、Solarized 等等——你只需改动一个样式表链接就能切换。主题就是 CSS，所以所谓"自定义 reveal.js 主题"，意思是写（或 fork）一个 Sass/CSS 文件，去设定你的字体、颜色和标题样式。这里没有主题<em>商店</em>；你是在编辑样式表。</p>
      <p><strong>reveal.js 模板和示例</strong>大多存在于社区，而不是某个官方陈列馆。仓库自带示例幻灯片，文档站（revealjs.com）本身就是一套你可以查看源码的 reveal.js 幻灯片，GitHub 上更是满是入门模板、脚手架和框架封装。如果你在搜"reveal js examples"，信息量最高的就是官方站点自己的源码——它几乎把每个功能都就地演示了一遍。给新手的提醒：这里的"模板"是一个供你克隆并编辑的代码仓库，而不是一个你往里填内容的设计文件。</p>
      <h2>导出到 PowerPoint 与 PDF</h2>
      <p><strong>PDF 导出是一等公民、内建功能</strong>，而且表现很好。在你幻灯片的 URL 后面加上 <code>?print-pdf</code>，打开浏览器的打印对话框，选择"另存为 PDF"，reveal.js 就会把每一页幻灯片重排到各自的一页上。这是干净、受支持的路径——你拿到的 PDF 看起来和你的幻灯片一模一样。</p>
      <p><strong>把 reveal.js 导出到 PowerPoint，则是需要管理预期的地方。</strong>并没有一个原生的"另存为 .pptx"能产出可编辑的 PowerPoint 文件。可行的路子是 PDF → PPTX 转换，而这种转换是<strong>有损的</strong>：每一页幻灯片进到 PowerPoint 里都是一张扁平的图片，而不是可编辑的文本框、形状和版式。所以如果下游有人需要在 PowerPoint 里打开你的幻灯片并<em>编辑</em>它——重打一个标题、给图表换个样式、重排一条要点——reveal.js 给不了他们这个。它天生就是用来从浏览器演示的，它可编辑的源是 HTML，而不是 <code>.pptx</code>。在动手之前就明白这一点，能让你在某天同事跟你要"那份 PowerPoint"时免于一场痛苦的意外。</p>
      <h2>reveal.js 的替代方案，以及它的对比</h2>
      <p>把幻灯片当代码来写，reveal.js 并不是唯一的路子。最接近的两个开源替代方案——也就是人们说"reveal.js 替代方案"时所指的——是 <strong>Slidev</strong> 和 <strong>Marp</strong>；而且实话说，Open Design 完全处在另一个层面，而不是与它们正面竞争。</p>

      <p><a href="/blog/slidev/"><strong>Slidev</strong></a> 是理念上最接近的兄弟：它以 Markdown 为先，底层配上 Vue 组件，因此它用 reveal.js 那种原始的 HTML 自由度，换来了一套更结构化、对开发者更友好的编写流程。如果你活在 Vue/Vite 的世界里、又想要实时代码演示，Slidev 是自然之选。<a href="/blog/marp/"><strong>Marp</strong></a> 则走了另一个方向——<em>只</em>用 Markdown、刻意极简、没有组件——这让它在你的幻灯片以文字为主、又完全不想操心框架时成为最快的路径。</p>
      <p>这三者都是代码优先的框架，有着相同的根本形态：你写出这套幻灯片，你在浏览器里演示，而可编辑的 PowerPoint 输出并不是重点。在它们之间做选择，主要看你想要哪种编写界面（原始 HTML、Vue 版 Markdown，还是纯 Markdown）——而不是某一个有、其余都缺的能力。</p>
      <h2>Open Design 的位置</h2>
      <p>这里说句实在的边界话。reveal.js 要求你去<strong>写</strong>这套幻灯片——每一页，都用代码。如果你想要手工雕琢的 HTML 控制力，这是个优点，而且如果这就是你的目标，这里没有任何东西能取代它。</p>
      <p><a href="/">Open Design</a> 是另一个层面，而且它是我们做的产品，所以请带着这一点来读。它是一个开源（Apache-2.0）、本地优先、自备 key 的 <strong>Agent-Native Design Workspace</strong>，它位于你已经在用的编程 Agent——Claude Code、Codex 或 Cursor——<em>之外</em>。你不必手写 HTML，而是描述你想要的幻灯片，Agent 就对着一套设计系统生成一份可编辑的，于是它产出时就符合品牌调性，并且保持可编辑，而不是被冻结成一份已渲染的成品。Open Design 还提供了你可以直接上手的第一方<a href="/plugins/templates/">幻灯片模板</a>。</p>
      <p>如果你想要的就是把幻灯片当代码来写、并拥有完整的 CSS 控制，那么它不是 reveal.js 的替代品——那是 reveal.js 的主场，而它做得极好。Open Design 面向的是这样的情形：你宁愿给 Agent 下简报，也不想去写标记；你希望输出能在许多套幻灯片之间都符合同一套设计系统；你希望最后拿到归你所有的可编辑文件。当这套幻灯片<em>就是</em>你正在写的代码时，去用 reveal.js；当这套幻灯片只是一个 Agent 为你产出的许多份符合品牌调性的成果之一时，去找一个工作空间。</p>
      <h2>常见问题</h2>
      <p><strong>reveal.js 免费吗？</strong>免费。reveal.js 是 MIT 许可证下的开源项目——可免费商用，没有账号、没有水印、没有导出限制。你自行托管这份 HTML。</p>
      <p><strong>我怎么添加或更换 reveal.js 主题？</strong>reveal.js 自带内建主题（Black、White、League、Sky、Night、Serif 等等），你改动一个样式表链接就能切换。自定义主题意味着写或 fork 一个带有你自己字体和颜色的 CSS/Sass 文件——这里没有可供浏览的主题商店。</p>
      <p><strong>我能把 reveal.js 导出成 PowerPoint 吗？</strong>导不成可编辑的 <code>.pptx</code>。reveal.js 能干净地导出成 PDF（加上 <code>?print-pdf</code>，然后从浏览器打印成 PDF）。把那份 PDF 转成 PowerPoint 是可以的，但有损——每一页都落成一张扁平图片，而不是可编辑的文字和形状。如果你需要一份可编辑的 PowerPoint，reveal.js 不是对的源头。</p>
      <p><strong>reveal.js、Slidev、Marp——我该用哪个？</strong>三者都是开源、代码优先、MIT 许可证。想要原始 HTML 控制选 reveal.js，想要 Markdown 加 Vue 组件（很适合实时演示）选 Slidev，想要最简单的纯 Markdown 幻灯片选 Marp。它们都不以可编辑的 PowerPoint 输出为目标。</p>
      <p><strong>用 reveal.js 需要会写代码吗？</strong>基本上，需要。reveal.js 是一个面向开发者的框架——你在文本编辑器里跟 HTML、Markdown 和 CSS 打交道，而不是一个拖拽式的图形界面。如果你更想用大白话描述一套幻灯片，那正是像 Open Design 这样的 agent-native 工作空间填补的空白。</p>
      <h2>结语</h2>
      <p>reveal.js 配得上它的 71.9k 星标：它是一种免费、MIT 许可证、HTML 原生的方式，能做出在任何有浏览器的地方都能运行的幻灯片，带有真正的主题、干净的 PDF 导出，以及完整的 CSS 控制。只是要看清了再进场——它是一个你在里面写代码的框架，而不是一个你点着走的应用，而且"导出成可编辑的 PowerPoint"并不在它的能力之内。如果你想把幻灯片当代码来写，reveal.js（或 <a href="/blog/slidev/">Slidev</a> / <a href="/blog/marp/">Marp</a>）是个很好的归宿。如果你更愿意描述一套幻灯片、让你的 Agent 生成一份可编辑、符合品牌调性、且归你所有的成果，那就该由一个 <a href="/">Agent-Native Design Workspace</a> 来接手了。</p>

      <h2>reveal.js とは</h2>
      <p>reveal.js は自らを「The HTML Presentation Framework」と称しており、その説明は的確です。開いてクリックして操作するアプリではなく、HTML ファイルに組み込む JavaScript ライブラリです。各スライドは <code>&lt;section&gt;</code> 要素で、section を入れ子にすると縦方向のスライドになります。すべてを CSS でスタイリングし、data 属性でトランジションやフラグメントを組み込み、ブラウザでページを開いて発表します。出力が<em>まさに</em> Web ページなので、Web でできること——ライブコード、埋め込み動画、iframe、WebGL、あなた自身のコンポーネント——は何でもスライド上に置けます。</p>
      <p>トレードオフは定義そのものに表れています。これは<strong>開発者向けのフレームワークであって、GUI ではありません</strong>。あなたのツールはコードエディタとブラウザであって、ドラッグハンドル付きのキャンバスではありません。だからこそ開発者に愛されるのです——完全なバージョン管理、完全な CSS 制御、独自ファイル形式なし——そしてだからこそ、コードを書かない人ははじき返されます。しかも本当に無料です。アカウント不要、ウォーターマークなし、エクスポート上限なし、席数課金なし。HTML はあなたのものです。</p>
      <h2>テーマ、テンプレート、サンプル</h2>
      <p>reveal.js まわりの検索の多くは、実のところ、ゼロからデザインせずに<em>見栄えよく</em>するにはどうするか、という話です。というわけで、各要素がどう組み合わさるのかを説明します。</p>
      <p><strong>reveal.js のテーマ</strong>は組み込みの出発点です。フレームワークには既製のテーマ一式——Black、White、League、Beige、Sky、Night、Serif、Simple、Solarized など——が付属し、スタイルシートのリンクを 1 か所変えるだけで切り替えられます。テーマは単なる CSS なので、「カスタム reveal.js テーマ」とは、フォント・色・見出しスタイルを設定する Sass/CSS ファイルを書く（あるいはフォークする）ことを意味します。テーマ<em>ストア</em>はありません。あなたが編集するのはスタイルシートです。</p>
      <p><strong>reveal.js のテンプレートやサンプル</strong>は、単一の公式ギャラリーではなく、主にコミュニティの中にあります。リポジトリにはサンプルのデッキが同梱され、ドキュメントサイト（revealjs.com）自体がソースを読める reveal.js のデッキであり、GitHub にはスターターテンプレート、ボイラープレート、フレームワークのラッパーがあふれています。「reveal js examples」を探しているなら、最も情報量が多いのは公式サイト自身のソースです——ほぼすべての機能を実地でデモしています。初心者向けの注意点：ここでの「テンプレート」は、内容を埋めていくデザインファイルではなく、クローンして編集するコードリポジトリです。</p>
      <h2>PowerPoint と PDF へのエクスポート</h2>
      <p><strong>PDF エクスポートは一級市民の組み込み機能</strong>で、きちんと機能します。デッキの URL に <code>?print-pdf</code> を付け、ブラウザの印刷ダイアログを開いて「PDF として保存」を選ぶと、reveal.js が各スライドをそれぞれ 1 ページに整えます。これがクリーンでサポートされた経路であり、得られる PDF はスライドそのままの見た目になります。</p>
      <p><strong>reveal.js を PowerPoint にエクスポートするところは、期待値の管理が必要な領域です。</strong>編集可能な PowerPoint ファイルを生成するネイティブの「.pptx として保存」はありません。現実的な経路は PDF → PPTX 変換で、その変換は<strong>非可逆</strong>です。各スライドは PowerPoint 上で、編集可能なテキストボックス・図形・レイアウトではなく、フラットな画像として届きます。ですから、後工程で誰かがあなたのデッキを PowerPoint で開いて<em>編集</em>する必要がある場合——見出しを打ち直す、グラフのスタイルを変える、箇条書きを組み替える——reveal.js はそれを与えてくれるツールではありません。ブラウザから発表するために作られており、その編集可能なソースは <code>.pptx</code> ではなく HTML です。始める前にこれを知っておけば、同僚に「あの PowerPoint」を求められた日の痛い驚きを避けられます。</p>

      <p>スライドをコードとして書く方法は reveal.js だけではありません。最も近いオープンソースの代替——人々が「reveal.js の代替」と言うときに指すもの——は <strong>Slidev</strong> と <strong>Marp</strong> です。そして正直に言えば、Open Design は真っ向から競合するのではなく、まったく別のレイヤーに位置しています。</p>

      <p><a href="/blog/slidev/"><strong>Slidev</strong></a> は思想的に最も近い兄弟です。Markdown を主体に据え、その下に Vue コンポーネントを敷くことで、reveal.js の生の HTML の自由度をいくらか手放す代わりに、より構造化され開発者にとって扱いやすい執筆フローを得ています。Vue/Vite の世界に住んでいて、ライブコーディングのデモをしたいなら、Slidev が自然な選択です。<a href="/blog/marp/"><strong>Marp</strong></a> は逆方向へ進みます——<em>Markdown のみ</em>、意図的にミニマル、コンポーネントなし——これにより、スライドがほぼテキストで、フレームワークのことを一切考えたくないときに最速の経路になります。</p>
      <p>3 つとも同じ根本形状を持つコードファーストのフレームワークです。デッキを書き、ブラウザで発表し、編集可能な PowerPoint 出力は本質的な狙いではありません。これらの間で選ぶことは、主にどの執筆面を望むか（生の HTML、Vue の Markdown、素の Markdown）の問題であって、どれか 1 つにあって他にない機能の問題ではありません。</p>
      <h2>Open Design が位置づくところ</h2>
      <p>ここで率直な境界線を引きます。reveal.js はデッキを<strong>書く</strong>ことを求めます——すべてのスライドを、コードで。手作りの HTML 制御が欲しいならこれは長所であり、それが目的ならここにあるどれもそれを置き換えません。</p>
      <p><a href="/">Open Design</a> は別のレイヤーであり、しかも私たちが作っている製品なので、その前提で読んでください。これはオープンソース（Apache-2.0）、ローカルファースト、キー持ち込み方式の <strong>Agent-Native Design Workspace</strong> で、あなたがすでに使っているコーディングエージェント——Claude Code、Codex、Cursor——の<em>外側</em>に位置します。HTML を手書きする代わりに、欲しいデッキを説明すると、エージェントがデザインシステムに沿って編集可能なものを生成します。だから最初からブランドに沿って仕上がり、レンダリング済みの成果物として凍りつくのではなく、編集可能なまま保たれます。Open Design は、そこから始められるファーストパーティの<a href="/plugins/templates/">デッキテンプレート</a>も提供しています。</p>
      <p>完全な CSS 制御でスライドをコードとして書きたいなら、Open Design は reveal.js の代替にはなりません——そこは reveal.js の本拠地であり、そこでの出来は見事です。Open Design が向いているのは、マークアップを書くよりエージェントにブリーフを渡したい、出力を多数のデッキにわたって 1 つのデザインシステムに合わせたい、最後に自分が所有する編集可能なファイルが欲しい、というケースです。デッキ<em>そのもの</em>があなたの書くコードであるときは reveal.js に手を伸ばし、デッキがエージェントがあなたのために生み出す多くのブランド準拠の出力の 1 つであるときはワークスペースに手を伸ばしてください。</p>
      <h2>FAQ</h2>
      <p><strong>reveal.js は無料ですか？</strong> はい。reveal.js は MIT ライセンスのオープンソースです——商用利用も無料で、アカウント不要、ウォーターマークなし、エクスポート制限なし。HTML は自分でホストします。</p>
      <p><strong>reveal.js のテーマはどう追加・変更しますか？</strong> reveal.js には組み込みテーマ（Black、White、League、Sky、Night、Serif など）が付属し、スタイルシートのリンクを 1 か所変えるだけで入れ替えられます。カスタムテーマとは、自分のフォントと色を持つ CSS/Sass ファイルを書くかフォークすることを意味します——閲覧できるテーマストアはありません。</p>
      <p><strong>reveal.js を PowerPoint にエクスポートできますか？</strong> 編集可能な <code>.pptx</code> にはできません。reveal.js は PDF へはきれいにエクスポートできます（<code>?print-pdf</code> を付けて、ブラウザから PDF に印刷します）。その PDF を PowerPoint に変換することは可能ですが非可逆です——各スライドは編集可能なテキストや図形ではなく、フラットな画像として落ちます。編集可能な PowerPoint が必要なら、reveal.js は適切なソースではありません。</p>
      <p><strong>reveal.js と Slidev と Marp——どれを使うべき？</strong> 3 つともオープンソース、コードファースト、MIT ライセンスです。生の HTML 制御には reveal.js、Markdown と Vue コンポーネント（ライブデモに最適）には Slidev、最もシンプルな Markdown のみのデッキには Marp を選びましょう。どれも編集可能な PowerPoint 出力は狙っていません。</p>
      <p><strong>reveal.js を使うにはコードが書けないといけませんか？</strong> 実質的には、はい。reveal.js は開発者向けのフレームワークです——ドラッグ＆ドロップの GUI ではなく、テキストエディタで HTML、Markdown、CSS を扱います。代わりに平易な言葉でデッキを説明したいなら、それこそ Open Design のような agent-native なワークスペースが埋める空白です。</p>
      <h2>まとめ</h2>
      <p>reveal.js は 71.9k のスターにふさわしい存在です。無料で、MIT ライセンスで、HTML ネイティブに、ブラウザがある場所ならどこでも動くスライドを作れ、本物のテーマ、クリーンな PDF エクスポート、完全な CSS 制御を備えています。ただし目を開いて臨んでください——これはあなたが中で書くフレームワークであって、クリックして進むアプリではなく、「編集可能な PowerPoint へのエクスポート」はやってくれることではありません。デッキをコードとして書きたいなら、reveal.js（あるいは <a href="/blog/slidev/">Slidev</a> / <a href="/blog/marp/">Marp</a>）は素晴らしい住処です。デッキを説明して、編集可能でブランドに沿った、自分が所有する成果をエージェントに生成させたいなら、そこで <a href="/">Agent-Native Design Workspace</a> が引き継ぎます。</p>

C'est aussi vraiment gratuit : pas de compte, pas de filigrane, pas de plafond d'export, pas de tarif au siège. Le HTML vous appartient.</p>
      <h2>Thèmes, modèles et exemples</h2>
      <p>La plupart des recherches autour de reveal.js visent en réalité à le rendre <em>beau</em> sans partir de zéro, alors voici comment les pièces s'emboîtent.</p>
      <p>Les <strong>thèmes reveal.js</strong> sont le point de départ intégré. Le framework est livré avec un ensemble de thèmes prêts à l'emploi — Black, White, League, Beige, Sky, Night, Serif, Simple, Solarized et d'autres — que vous changez en modifiant un seul lien de feuille de style. Un thème n'est que du CSS, donc un « thème reveal.js personnalisé » signifie écrire (ou forker) un fichier Sass/CSS qui définit vos polices, vos couleurs et vos styles de titres. Il n'y a pas de <em>boutique</em> de thèmes ; vous éditez des feuilles de style.</p>
      <p>Les <strong>modèles et exemples reveal.js</strong> vivent surtout dans la communauté plutôt que dans une galerie officielle unique. Le dépôt fournit des présentations d'exemple, le site de documentation (revealjs.com) est lui-même une présentation reveal.js dont vous pouvez lire la source, et GitHub regorge de modèles de démarrage, de boilerplates et de wrappers de framework. Si vous cherchez « reveal js examples », le plus riche en signal est la source du site officiel lui-même — il démontre presque chaque fonctionnalité en situation. Le piège pour les débutants : un « modèle » est ici un dépôt de code à cloner et à éditer, pas un fichier de design à remplir.</p>
      <h2>Export vers PowerPoint et PDF</h2>
      <p><strong>L'export PDF est une fonctionnalité de premier ordre, intégrée</strong>, et elle fonctionne bien. Ajoutez <code>?print-pdf</code> à l'URL de votre présentation, ouvrez la boîte de dialogue d'impression de votre navigateur, choisissez « Enregistrer au format PDF », et reveal.js réagence chaque diapositive sur sa propre page. C'est la voie propre et prise en charge — le PDF obtenu ressemble exactement à vos diapositives.</p>
      <p><strong>Exporter reveal.js vers PowerPoint, c'est là qu'il faut gérer les attentes.</strong> Il n'existe pas d'« Enregistrer sous .pptx » natif qui produise un fichier PowerPoint modifiable. La voie pratique est la conversion PDF → PPTX, et cette conversion est <strong>avec perte</strong> : chaque diapositive arrive dans PowerPoint sous forme d'image plate, et non de zones de texte, de formes et de mises en page modifiables. Donc si, en aval, quelqu'un doit ouvrir votre présentation dans PowerPoint et la <em>modifier</em> — retaper un titre, restyler un graphique, réagencer une puce — reveal.js n'est pas l'outil qui le lui permet. Il est conçu pour être présenté depuis le navigateur, et sa source modifiable est le HTML, pas le <code>.pptx</code>. Le savoir avant de commencer vous épargne une mauvaise surprise le jour où un collègue réclame « le PowerPoint ».</p>

      <h2>La place d'Open Design</h2>
      <p>Voici la frontière honnête. reveal.js vous demande d'<strong>écrire</strong> la présentation — chaque diapositive, dans le code. C'est un atout si vous voulez un contrôle HTML fait main, et rien ici ne le remplace si c'est votre objectif.</p>
      <p><a href="/">Open Design</a> est une autre couche, et c'est un produit que nous concevons, alors lisez ceci en le gardant à l'esprit. C'est un <strong>Agent-Native Design Workspace</strong> open source (Apache-2.0), local-first et bring-your-own-key, qui se place <em>en dehors</em> de l'agent de code que vous utilisez déjà — Claude Code, Codex ou Cursor. Au lieu de rédiger le HTML à la main, vous décrivez la présentation voulue et l'agent en génère une version modifiable appuyée sur un design system, si bien qu'elle sort conforme à la marque et reste modifiable plutôt que figée en artefact rendu. Open Design fournit aussi ses propres <a href="/plugins/templates/">modèles de présentation</a> pour démarrer.</p>
      <p>Ce n'est pas un remplacement de reveal.js si ce que vous voulez, c'est écrire des diapositives sous forme de code avec un contrôle CSS total — c'est le terrain de reveal.js, et il y excelle. Open Design est pour le cas où vous préférez briefer un agent plutôt qu'écrire du balisage, où vous voulez que la sortie corresponde à un design system sur de nombreuses présentations, et où vous voulez, au bout du compte, des fichiers modifiables qui vous appartiennent. Tournez-vous vers reveal.js quand la présentation <em>est</em> le code que vous écrivez ; tournez-vous vers un espace de travail quand la présentation est l'une des nombreuses sorties conformes à la marque qu'un agent produit pour vous.</p>

If you write code and you need to give a talk, you have probably hit **reveal.js**. It's the open-source framework behind a huge share of the developer-conference slides you've watched: you author each slide in HTML or Markdown, and it runs as a web page in any browser — with themes, transitions, fragment animations, speaker notes, and native PDF export built in. With 71.9k stars on GitHub and an MIT license, it's the default answer to "how do I make slides without opening PowerPoint."

This guide is an honest map of it. What reveal.js actually is, how themes and templates and examples fit together, how exporting to PDF and PowerPoint really behaves (there's a catch), how it stacks up against the other open-source contenders, and — the part most write-ups skip — where a framework like this stops being the right tool and something else takes over. If you're weighing your options for an [AI-and-agent-assisted deck workflow](/blog/claude-ppt-skills/), it helps to know exactly what reveal.js is and isn't.

## What reveal.js is

reveal.js calls itself "The HTML Presentation Framework," and that description is precise. It is not an app you open and click around in — it's a JavaScript library you drop into an HTML file. Each slide is a `<section>` element; nest sections and you get vertical slides. You style everything with CSS, wire in transitions and fragments with data-attributes, and present by opening the page in a browser. Because the output *is* a web page, anything the web can do — live code, embedded video, iframes, WebGL, your own components — can live on a slide.

The trade-off is right there in the definition: this is a **developer framework, not a GUI**. Your tools are a code editor and a browser, not a canvas with drag handles. That's exactly why developers love it — full version control, full CSS control, no proprietary file format — and exactly why a non-coder will bounce off it. It's also genuinely free: no account, no watermark, no export cap, no per-seat pricing. You own the HTML.

## Themes, templates & examples

Most searches around reveal.js are really about making it *look* good without designing from scratch, so here's how the pieces fit.

**reveal.js themes** are the built-in starting point. The framework ships with a set of ready-made themes — Black, White, League, Beige, Sky, Night, Serif, Simple, Solarized, and more — that you switch by changing a single stylesheet link. A theme is just CSS, so a "custom reveal.js theme" means writing (or forking) a Sass/CSS file that sets your fonts, colors, and heading styles. There's no theme *store*; you're editing stylesheets.

**reveal.js templates and examples** live mostly in the community rather than in one official gallery. The repository ships example decks, the docs site (revealjs.com) is itself a reveal.js deck you can read the source of, and GitHub is full of starter templates, boilerplates, and framework wrappers. If you're searching for "reveal js examples," the highest-signal one is the official site's own source — it demonstrates nearly every feature in place. The catch for beginners: a "template" here is a code repository to clone and edit, not a design file you fill in.

## Exporting to PowerPoint & PDF

**PDF export is a first-class, built-in feature** and it works well. Append `?print-pdf` to your deck's URL, open your browser's print dialog, choose "Save as PDF," and reveal.js reflows every slide onto its own page. This is the clean, supported path — the PDF you get looks exactly like your slides.

**Exporting reveal.js to PowerPoint is where expectations need managing.** There is no native "Save as .pptx" that produces an editable PowerPoint file. The practical route is PDF → PPTX conversion, and that conversion is **lossy**: each slide arrives in PowerPoint as a flat image, not as editable text boxes, shapes, and layouts. So if someone downstream needs to open your deck in PowerPoint and *edit* it — retype a headline, restyle a chart, reflow a bullet — reveal.js is not the tool that gives them that. It's built to be presented from the browser, and its editable source is HTML, not `.pptx`. Knowing this before you start saves a painful surprise the day a colleague asks for "the PowerPoint."

## reveal.js alternatives & how it compares

reveal.js isn't the only way to write slides as code. The two closest open-source alternatives — the ones people mean by "reveal.js alternative" — are **Slidev** and **Marp**, and it's worth being honest that Open Design sits in a different layer entirely rather than competing head-on.

| Tool | Authoring | Editable .pptx | Templating | License |
| --- | --- | --- | --- | --- |
| **reveal.js** | HTML or Markdown | No (PDF→PPTX is lossy images) | Themes = CSS/Sass; community templates | MIT |
| **Slidev** | Markdown + Vue | No (image/PDF export) | Vue components + themes | MIT |
| **Marp** | Markdown only | No (PDF/PPTX as images) | Markdown themes (CSS) | MIT |
| **Open Design** | Describe it to your agent | Yes — editable output against a design system | First-party deck templates | Apache-2.0 |

[**Slidev**](/blog/slidev/) is the closest philosophical sibling: it's Markdown-first with Vue components underneath, so it trades some of reveal.js's raw HTML freedom for a more structured, developer-ergonomic authoring flow. If you live in the Vue/Vite world and want live coding demos, Slidev is the natural pick. [**Marp**](/blog/marp/) goes the other direction — Markdown *only*, deliberately minimal, no components — which makes it the fastest path when your slides are mostly text and you never want to think about a framework.

All three are code-first frameworks with the same fundamental shape: you write the deck, you present in a browser, and editable-PowerPoint output isn't really the point. Choosing between them is mostly about which authoring surface you want (raw HTML, Vue Markdown, or plain Markdown) — not about capabilities that one has and the others lack.

## Where Open Design fits

Here's the honest boundary. reveal.js asks you to **write** the deck — every slide, in code. That's a feature if you want hand-crafted HTML control, and nothing here replaces it if that's your goal.

[Open Design](/) is a different layer, and it's a product we make, so read this with that in mind. ## FAQ

**Is reveal.js free?** Yes. reveal.js is open source under the MIT license — free to use commercially, with no account, no watermark, and no export limits. You self-host the HTML.

**How do I add or change a reveal.js theme?** reveal.js ships built-in themes (Black, White, League, Sky, Night, Serif, and more) that you swap by changing one stylesheet link. A custom theme means writing or forking a CSS/Sass file with your own fonts and colors — there's no theme store to browse.

**Can I export reveal.js to PowerPoint?** Not to an editable `.pptx`. reveal.js exports cleanly to PDF (add `?print-pdf` and print to PDF from your browser). Converting that PDF to PowerPoint is possible but lossy — each slide lands as a flat image, not editable text and shapes. If you need an editable PowerPoint, reveal.js isn't the right source.

**reveal.js vs Slidev vs Marp — which should I use?** All three are open-source, code-first, MIT-licensed. Pick reveal.js for raw HTML control, Slidev for Markdown-plus-Vue components (great for live demos), and Marp for the simplest Markdown-only decks. None targets editable-PowerPoint output.

**Do I need to know how to code to use reveal.js?** Effectively, yes. reveal.js is a developer framework — you work in a text editor with HTML, Markdown, and CSS, not a drag-and-drop GUI. If you want to describe a deck in plain language instead, that's the gap an agent-native workspace like Open Design fills.

reveal.js earns its 71.9k stars: it's a free, MIT-licensed, HTML-native way to build slides that run anywhere a browser does, with real themes, clean PDF export, and total CSS control. Just go in clear-eyed — it's a framework you write in, not an app you click through, and "export to editable PowerPoint" isn't something it does. If you want to author decks as code, reveal.js (or [Slidev](/blog/slidev/) / [Marp](/blog/marp/)) is a great home.
