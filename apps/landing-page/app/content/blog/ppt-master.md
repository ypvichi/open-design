      <h2>ppt-master 是什么</h2>
      <p>ppt-master（GitHub <code>hugohe3/ppt-master</code>，MIT 许可证，主页在 <a href="https://hugohe3.github.io/ppt-master/">hugohe3.github.io/ppt-master</a>）是一个由 agent 驱动的工具，把文档转换成可编辑的 PowerPoint 文件。你喂给它素材——笔记、一份报告、一份规格说明、一份大纲——它就运行一套 agent 工作流，规划整套幻灯片、排布页面，并输出一个真正的 <code>.pptx</code>。它和编程 agent 的 PPT 技能一同处在开源、agent-native 的幻灯片领域里，但它的形态更像一个应用或工作流，而不是一个即插即用的单一技能。</p>

      <h2>带原生形状的可编辑 .pptx——差异所在</h2>
      <p>这里正是把 ppt-master 与大多数同类拉开距离的一条线。很多"AI PPT 生成器"导出的幻灯片<em>看起来</em>像 PowerPoint，其实只是图片——每页一张渲染好的图，或者一个你在浏览器里演示的 HTML 页面。你选不中标题、重打不了要点、也挪不动一个形状；你只能重新生成。</p>
      <p>ppt-master 产出的是真东西：一个由<strong>原生 PowerPoint 对象</strong>搭起来的 <code>.pptx</code>——文本框、形状和动画，都是 PowerPoint 视为一等公民的元素。打开它，每一部分都可编辑，任何人都能改，不需要写代码，也不需要特殊应用。当你的交付物必须是一个非开发者也能改动、而且是<em>在 PowerPoint 里</em>改动的文件时，这种真正的可编辑性，正是你选择 ppt-master 的全部理由。</p>

      <h2>诚实的说明</h2>
      <p>有几件事值得说清楚。ppt-master 采用 <strong>MIT 许可证</strong>，这大概是开源里最宽松的一档——如果你哪天想在它之上做点东西，或把它并进自己的工具链，会很友好。而且它更像一个<strong>应用/工作流，而非单一"技能"</strong>：它不是一个你克隆进编程 agent 技能目录的单文件 <code>SKILL.md</code>，而是一条你去运行的"文档到幻灯片"流水线。这不是贬低——这正是它这份活儿该有的形态——但它的搭建方式和心智模型跟插件式技能不同，所以请据此校准预期。</p>

      <h2>Open Design 的位置</h2>
      <p>ppt-master 是为一件事量身打造的——文档进，可编辑的 <code>.pptx</code> 出——当那个原生可编辑的文件是硬性要求时，它是最强的开源选择。它在这件事上是真的好，本文也无意劝你换别的来做这件事。</p>
      <p><a href="/">Open Design</a> 站在更宽一层。它是一个开源（Apache-2.0）、本地优先、自备 key 的 <strong>Agent-Native Design Workspace</strong>——不是一个单一的"文档转幻灯片"转换器，而是一个让幻灯片和设计工作对着一套设计系统、始终符合品牌调性的地方，且贯穿你已经在用的编程 agent。ppt-master 把<em>一份文档</em>变成<em>一套幻灯片</em>，而 Open Design 面对的是这样的场景：这套幻灯片只是众多产物之一，它们都得看起来像出自同一个团队。它在<a href="/plugins/templates/">插件库</a>里提供第一方的 HTML 幻灯片模板，并且接进你现有的 agent，而不是另做一个独立应用。当你需要从一份文档里得到一个真正可编辑的 PowerPoint 时，用 ppt-master；当这套幻灯片必须保持一致、并与你其余的设计工作放在一起时，去找一个工作空间。</p>
      <h2>常见问题</h2>
      <p><strong>ppt-master 能做出可编辑的 PowerPoint 吗？</strong>能——这正是它的全部意义。它输出一个由原生 PowerPoint 对象（形状、文本、动画）搭成的真正 <code>.pptx</code>，所以任何人都能在 PowerPoint 里打开并编辑，无需写代码。这不同于那些把幻灯片导成图片、或只给浏览器 HTML 的工具。</p>
      <p><strong>ppt-master 免费且开源吗？</strong>是的。它以 MIT 许可证开源——业界最宽松的许可证之一——且免费使用。你自备模型的 key。</p>
      <p><strong>ppt-master 是一个 Claude 技能吗？</strong>不完全是。相比 frontend-slides 或 dashiAI 那种克隆即用的单一技能，它更像一个由 agent 驱动的应用或工作流。它处在同一个 agent-native、开源的 PPT 领域里，但你是把它当作一条"文档到幻灯片"的流水线来运行。</p>
      <p><strong>ppt-master 和其他 AI PPT 生成器有什么不同？</strong>它产出的是真正可编辑、带原生形状和动画的 <code>.pptx</code> 文件，而不是把幻灯片做成图片或不可编辑的 HTML。如果原生 PowerPoint 的可编辑性是你的硬性要求，这就是它的差异所在。</p>
      <p><strong>我该用 ppt-master 还是设计工作空间？</strong>当交付物是从一份文档得到的单个可编辑 PowerPoint 时，用 ppt-master。当这套幻灯片需要对着一套设计系统保持品牌调性、并与你其余的设计工作放在一起时，用像 Open Design 这样的 agent-native 工作空间。</p>

      <h2>ネイティブ図形を備えた編集可能な .pptx——差別化要素</h2>
      <p>ここが、ppt-master をほとんどの同種ツールから隔てる一線です。多くの「AI PPT ジェネレーター」がエクスポートするスライドは、PowerPoint の<em>ように見えて</em>、実は画像です——スライド 1 枚ごとにレンダリングされた 1 枚の絵か、ブラウザで発表する HTML ページなのです。タイトルを選択することも、箇条書きを打ち直すことも、図形をずらすこともできません。できるのは再生成だけです。</p>
      <p>ppt-master が生み出すのは本物です。<strong>ネイティブの PowerPoint オブジェクト</strong>——PowerPoint が第一級の要素として理解するテキストボックス、図形、アニメーション——から組み立てられた <code>.pptx</code> です。開けばどの要素も編集可能で、誰でも、コードも専用アプリも要らずに手を加えられます。成果物が、非開発者でもなお変更できる——しかも<em>PowerPoint の中で</em>変更できる——ファイルでなければならないとき、その真の編集可能性こそが、ppt-master に手を伸ばす理由のすべてです。</p>

      <h2>FAQ</h2>

"Can I turn this doc into a PowerPoint I can actually edit?" is a fair thing to want, and most AI slide tools quietly fail it — they hand you a picture of a deck, or a browser page you present but can't drop into PowerPoint. [ppt-master](https://github.com/hugohe3/ppt-master) is the open-source project that takes the question literally. Point it at a document and an agent workflow produces a genuine `.pptx` with native shapes and animations — the kind of file a colleague can open in PowerPoint and move a box around in.

That single property is why it has drawn 38k GitHub stars and why it belongs in the same conversation as the [Claude PPT skills](/blog/claude-ppt-skills/). This is an honest guide to what ppt-master is, how it works, where its differentiator actually matters, and where a broader design workspace takes over when the deck needs to outlive one prompt.

## What ppt-master is

ppt-master (GitHub `hugohe3/ppt-master`, MIT-licensed, homepage at [hugohe3.github.io/ppt-master](https://hugohe3.github.io/ppt-master/)) is an agent-driven tool that converts documents into editable PowerPoint files. You feed it source material — notes, a report, a spec, an outline — and it runs an agent workflow that plans the deck, lays out slides, and emits a real `.pptx`. It sits alongside the coding-agent PPT skills in the open-source, agent-native slide space, but it's shaped more like an app or workflow than a single drop-in skill.

## Editable .pptx with native shapes — the differentiator

Here's the line that separates ppt-master from most of the field. A lot of "AI PPT generator" tools export slides that *look* like PowerPoint but are really images — a rendered picture per slide, or an HTML page you present in a browser. You can't select the title, retype a bullet, or nudge a shape; you can only regenerate.

ppt-master produces the real thing: a `.pptx` built from **native PowerPoint objects** — text boxes, shapes, and animations that PowerPoint understands as first-class elements. Open it and every piece is editable, by anyone, no code and no special app required. When your deliverable has to be a file a non-developer can still change — and change *in PowerPoint* — that genuine editability is the whole reason to reach for ppt-master.

## How it works

The flow is short:

1. **Give it a document** — notes, a report, an outline, or any source text you want turned into a deck.
2. **The agent workflow runs** — it plans the narrative, structures slides, and lays out native shapes, text, and animations.
3. **You get an editable `.pptx`** — a real PowerPoint file with native objects, ready to open and edit like any deck you built by hand.

## Honest notes

A couple of things worth being straight about. ppt-master is **MIT-licensed**, which is about as permissive as open source gets — friendly if you ever want to build on it or fold it into your own tooling. And it's more of an **app/workflow than a single "skill"**: rather than a one-file `SKILL.md` you clone into a coding agent's skills directory, it's a document-to-deck pipeline you run. That's not a knock — it's the right shape for its job — but it's a different setup and mental model than the plug-in skills, so calibrate expectations accordingly.

## Where Open Design fits

ppt-master is purpose-built for one job — document in, editable `.pptx` out — and it's the strongest open-source option when that native-editable file is the hard requirement. It's genuinely good at that, and this isn't a nudge to use something else for it.

[Open Design](/) sits a layer wider. Where ppt-master turns *a document* into *a deck*, Open Design is for the case where the deck is one artifact among many that all have to look like they came from the same team.
