      <p>"编程 Agent 能做出好看的幻灯片吗？"这个问题，大多数时候是靠基于 HTML 的 skill 来回答的——它们把幻灯片渲染成网页。<strong>codex-ppt-skill</strong> 走的是另一条路：它把一个图像模型交给 Agent，让它把每一页幻灯片当作一张图来<em>设计</em>。最终得到的是一套每一页都是渲染图像的演示——封面美术、全出血视觉、直接烙进构图里的大字号——而不是可编辑的文本框。</p>

      <h2>codex-ppt-skill 是什么</h2>
      <p><strong><a href="https://github.com/ningzimu/codex-ppt-skill">codex-ppt-skill</a></strong>（3.4k★，MIT）是一个编程 Agent skill，它把你的 Agent 与图像生成——具体来说是 <strong>GPT-Image-2</strong>——配对起来，产出图像主导的 PowerPoint 演示。Agent 不再去排布文字和形状，而是为每一页写一段美术指导，并把它作为一张设计好的图像生成出来。它是为 OpenAI Codex 打造的，遵循标准的编程 Agent skill 模式：你把它安装进 Agent 的 skill 目录，描述这套演示，然后让它运行。如果你想在安装前先看看输出风格，这里有一个<a href="https://ningzimu.github.io/codex-ppt-skill/">在线主页</a>。</p>
      <p>HTML 幻灯片 skill 为干净、可编辑的网页做优化，而 codex-ppt-skill 为视觉丰富度做优化。它在以封面见长、追求强冲击力的演示上大放异彩——发布会演示、主题演讲开场、路演叙事，任何"画面本身就是重点"的场合。</p>

      <h2>图像优先的幻灯片（GPT-Image-2）——它是什么意思，什么时候合适</h2>
      <p>图像优先的幻灯片正如其字面意思：整页幻灯片就是一张生成的图像。Agent 不会把标题、副标题和要点列表作为彼此独立的对象摆放，而是构图出一张单一的图像，其中版式、图像和排版由 GPT-Image-2 一起渲染完成。你得到的是一套开箱即看上去经过美术指导的演示，看不到任何模板拼接的接缝。</p>
      <p>对合适的任务，这是实打实的优势；对不合适的任务，则是实打实的局限：</p>
      <ul><li><strong>当视觉即信息时，它是对的。</strong>封面、章节分隔、高光时刻、氛围浓烈的叙事、以品牌见长的路演演示。当"做得惊艳"胜过"改起来方便"时，图像优先胜出。</li><li><strong>当内容密集或多变时，它是错的。</strong>数据表格、冗长的议程、你会逐行修改的演示，或者任何下周需要有人重新敲一遍的东西。烙进图里的文字是文字的一张照片，而不是你能编辑的文字。</li></ul>
      <p>如果你的大多数幻灯片都是你会不断改动的文字，那么 HTML 幻灯片 skill 或一个可编辑输出的工具更合适。如果你的演示成败全系于视觉，那么图像优先就是你选择这个 skill 的全部理由。</p>

      <h2>如何使用</h2>
      <p>流程遵循编程 Agent skill 惯常的形态：</p>
      <ol><li><strong>安装 skill</strong>——把 codex-ppt-skill 克隆或添加到你的编程 Agent 的 skill 目录里（仓库里有安装步骤）。自备你的模型 key，包括用于图像生成的 GPT-Image-2 访问权限。</li><li><strong>描述这套演示</strong>——给 Agent 提示词，说明主题、受众、语气和大致页数，再加上视觉方向（美术风格、色彩氛围、封面处理）。图像优先的演示会因具体的美术指导而回报你更好的结果。</li><li><strong>让它生成</strong>——Agent 为每一页写一段构图，并逐页通过 GPT-Image-2 渲染。</li><li><strong>迭代</strong>——用大白话提要求（"把封面做得更暗、更有电影感"、"用一张产品特写重新生成第 3 页"）。因为幻灯片就是图像，迭代意味着重新渲染，而不是微调文本框。</li><li><strong>导出</strong>——把这些图像组装成一套 PowerPoint 演示用于演讲。</li></ol>
      <h2>诚实提醒</h2>
      <p>在你把一套演示托付给这套工作流之前，有两件事要记住：</p>
      <ul><li><strong>基于图像意味着不是可编辑的文字。</strong>每一页都是一张渲染出来的图，所以你没法点进某一页去改一个错别字或重排一个段落——你要重新给提示词并重新生成。这对一套惊艳的一次性演示来说没问题，但对一份你要长期维护的文档来说很痛苦。它还意味着你对确切的文字位置只有有限的精细控制；由模型来决定。</li><li><strong>MIT 许可证。</strong>codex-ppt-skill 采用 MIT 许可证，因此使用和嵌入都很宽松——没有 copyleft 的附带条款。你持续的成本在于模型用量：在图像密集的演示上，GPT-Image-2 的生成会累积起来，所以要为渲染定价，而不只是为安装定价。</li></ul>

      <h2>常见问题</h2>
      <p><strong>codex-ppt-skill 是什么？</strong>它是一个开源（MIT）的编程 Agent skill，把你的 Agent 与 GPT-Image-2 配对，构建图像主导的 PowerPoint 演示——每一页都是一张设计好的图像，而不是要点列表。它是为 OpenAI Codex 打造的，运行在标准的 skill 模式里。</p>
      <p><strong>它和 frontend-slides 或其他 Claude PPT skill 有什么不同？</strong>大多数 PPT skill 把幻灯片渲染成可编辑的 HTML 或形状。codex-ppt-skill 把每一页渲染成一张生成的图像，用可编辑性换取视觉丰富度。当演示的冲击力来自视觉、而非来自你会不断修改的文字时，它是最佳选择。</p>
      <p><strong>codex-ppt-skill 生成的幻灯片可编辑吗？</strong>作为文字不可编辑。每一页都是一张渲染出来的图像，所以你通过重新给提示词并重新生成来修改它，而不是编辑文本框。如果你需要逐行编辑，请改用 HTML 幻灯片 skill 或一个可编辑输出的工具。</p>
      <p><strong>codex-ppt-skill 免费吗？</strong>这个 skill 采用 MIT 许可证，可免费使用。你真正的成本在于模型用量——在图像密集的演示上，GPT-Image-2 的生成会累积起来——再加上你自己的 API key。</p>
      <p><strong>我什么时候该改用设计工作空间？</strong>当这套演示必须对着一套设计系统保持品牌调性、保持可编辑，并且与你其余的设计工作放在一起时。那正是像 Open Design 这样的 agent-native 工作空间从一个一次性 skill 手中接过来的地方。</p>
      <h2>结语</h2>
      <p>codex-ppt-skill 是当你想直接从终端里得到一套视觉丰富、以封面见长的演示，并且乐于让每一页都是一张设计好的图像而非可编辑文字时，值得选用的 skill。它采用 MIT 许可证、为编程 Agent 打造、由 GPT-Image-2 驱动——对高冲击力的一次性演示是一个干净利落的选择。而当这套演示反过来需要符合品牌调性、可编辑、并成为一项更大的设计工作的一部分时，那正是一个 <a href="/">Agent-Native Design Workspace</a> 接手的时刻：你的 Agent、你的文件，从提示词到交付的演示。</p>

      <p>これは、ふさわしい仕事には本物の強みであり、ふさわしくない仕事には本物の制約です。</p>

      <h2>Diapositives image d'abord (GPT-Image-2) — ce que cela signifie, et quand c'est justifié</h2>

L'agent ne place pas un titre, un sous-titre et une liste à puces comme des objets distincts — il compose une image unique où la mise en page, l'imagerie et la typographie sont rendues ensemble par GPT-Image-2.       <p>C'est une vraie force pour la bonne mission, et une vraie limite pour la mauvaise :</p>
      <ul><li><strong>Justifié quand le visuel est le message.</strong> Couvertures, séparateurs de sections, moments forts, récits chargés d'ambiance, présentations de pitch axées sur la marque. Quand « rendre saisissant » l'emporte sur « rendre facile à modifier », l'image d'abord gagne.</li><li><strong>À éviter quand le contenu est dense ou volatil.</strong> Tableaux de données, ordres du jour longs, présentations que vous réviserez ligne par ligne, ou tout ce qu'un décideur devra retaper la semaine prochaine. ## What codex-ppt-skill is

**[codex-ppt-skill](https://github.com/ningzimu/codex-ppt-skill)** (3.4k★, MIT) is a coding-agent skill that pairs your agent with image generation — specifically **GPT-Image-2** — to produce image-forward PowerPoint decks. Instead of laying out text and shapes, the agent writes an art direction for each slide and generates it as a designed image. It was built for OpenAI Codex and works in the standard coding-agent skill model: you install it into your agent's skill directory, describe the deck, and let it run. There's a [live homepage](https://ningzimu.github.io/codex-ppt-skill/) if you want to see the output style before installing.

Where an HTML-slide skill optimizes for clean, editable web pages, codex-ppt-skill optimizes for visual richness. It shines on cover-heavy, high-impact presentations — launch decks, keynote openers, pitch narratives, anything where the picture *is* the point.

## Image-first slides (GPT-Image-2) — what that means, and when it's right

An image-first slide is exactly what it sounds like: the whole slide is one generated image. The agent doesn't place a title, a subtitle and a bullet list as separate objects — it composes a single picture where the layout, imagery and typography are rendered together by GPT-Image-2. You get a deck that looks art-directed out of the box, with no template seams showing.

That's a real strength for the right job, and a real limitation for the wrong one:

- **Right when the visual is the message.** Covers, section breaks, hero moments, mood-heavy storytelling, brand-forward pitch decks. When "make it look striking" beats "make it easy to edit," image-first wins.
- **Wrong when the content is dense or volatile.** Data tables, long agendas, decks you'll revise line-by-line, or anything a stakeholder needs to retype next week. Baked-in text is a picture of words, not words you can edit.

If most of your slides are text you'll keep changing, an HTML-slide skill or an editable-output tool fits better. If your deck lives or dies on the visuals, image-first is the whole reason to reach for this skill.

## How to use it

The flow follows the usual coding-agent skill shape:

1. **Install the skill** — clone or add codex-ppt-skill to your coding agent's skill directory (the repo has setup steps). Bring your own model key, including GPT-Image-2 access for the image generation.
2. **Describe the deck** — prompt the agent with topic, audience, tone and rough slide count, plus the visual direction (art style, color mood, cover treatment). Image-first decks reward specific art direction.
3. **Let it generate** — the agent writes a per-slide composition and renders each one through GPT-Image-2.
4. **Iterate** — ask in plain language ("make the cover darker and more cinematic," "regenerate slide 3 with a product close-up"). Because slides are images, iteration means re-rendering, not nudging text boxes.
5. **Export** — assemble the images into a PowerPoint deck for presenting.

## Honest notes

Two things to keep in mind before you commit a deck to this workflow:

- **Image-based means not editable text.** Each slide is a rendered picture, so you can't click into a slide and fix a typo or reflow a paragraph — you re-prompt and regenerate. That's fine for a striking one-shot deck and painful for a document you'll maintain. It also means limited fine control over exact text placement; the model decides.
- **MIT license.** codex-ppt-skill is MIT-licensed, so it's permissive to use and embed — no copyleft strings. Your ongoing cost is model usage: GPT-Image-2 generations add up on image-heavy decks, so price the render, not just the install.
