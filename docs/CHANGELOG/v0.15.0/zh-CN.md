# Open Design 0.15.0 — 更低成本，更快交付：OD 的 DeepSeek 时刻

Open Design 0.15.0 通过优化整体 Design System Prompt，让日常设计任务更快、消耗更低：代表性评测中，首 token 时间缩短 **49.5%**，端到端时长缩短 **21.2%**，平均输入 token 消耗降低 **25.1%**。同时，从制作和演示 deck、通过 URL 复刻公开网站，到任务失败后的提示与恢复路径，整个创作流程也更加顺畅。

## ✨ 新增

### 🎞 Deck、演示与导出

- **Deck 终于像一个工作台，而不只是静态预览。** 多页 deck 支持缩略图导航、直接跳转页面，以及方向键、Home 和 End 键翻页，不必反复寻找控制按钮。

- **演讲者备注会留在它所属的那一页。** 可在当前幻灯片旁阅读和编辑备注、原地保存，并在演示者视图中查看备注，而不遮挡面向观众的画面。

- **按现场需要进行演示。** 可在当前标签页或全屏进入沉浸演示，并在不丢失 deck 状态的前提下上一页、下一页、暂停、继续或重置。

- **导出真正已确认的版本。** 可从指定文件版本下载 PDF、图片、ZIP 或 HTML，而不会悄悄导出最新修改；演讲者备注不会进入导出的幻灯片画面。

### 🌐 网站复刻

- **用一个 URL 开始网站复刻，而不是从空白 prompt 开始。** Website Clone 现已成为首页和资源库的一等入口：选择能力、粘贴公开 URL，Open Design 会用正确的上下文创建项目。

- **复刻过程有审计说明。** 工作流会先侦察页面结构、路由、资源与交互；生成项目中会保留 NOTES.md 风格的说明，记录方法、资源来源与已知差异。

- **让结果既好用也更负责任。** 生成结果适合本地预览，不会沿用第三方统计或广告脚本；面对登录墙等复杂目标时，会明确说明限制，而不是假装完整成功。

### 💬 动效与示例

- **把对话变成动效。** 新增 Chat Motion Overlay skill，可将双人对话转换为聊天动效，支持 WeChat、Telegram、Messenger 风格容器，以及用于后期叠加的透明输出选项。

- **在首页更快找到合适的起点。** Website Clone 拥有更清晰的入口、可直接使用的示例和以目标 URL 开头的 prompt；模板和插件卡片也更可靠地打开对应预览，再由你决定使用还是 Remix。

## 🔁 变化

### 🧠 任务、模型与集成

- **Design System Prompt 更轻，任务响应更快。** 通过优化整体 Design System Prompt，在代表性评测中，首 token 时间缩短 **49.5%**，端到端时长缩短 **21.2%**，平均输入 token 消耗降低 **25.1%**。这意味着日常设计任务响应更快、消耗更低，也能更高效地完成更多任务。

- **失败会说明发生了什么，以及下一步怎么做。** 未安装本地 agent、输入过长、模型或服务不可用、额度耗尽、超时、空输出、保存失败与工具循环，都有更具体的恢复路径。

- **任务完成意味着真正有交付物。** 没有生成或保存可用项目文件的运行，不再被展示为成功完成。

- **BYOK 和第三方 agent 配置更准确。** 新增 Atlas Cloud 预设；OpenAI-compatible endpoint 会走兼容的 chat-completions 路径；embedding 与 rerank 模型不再进入聊天模型选择器；Kiro、Reasonix、Antigravity、Grok Build 与 OpenRouter 的配置和运行问题也有更清楚的提示。

- **隐私选择更容易理解。** 同意与设置页面更明确地区分匿名运行指标和已脱敏的质量审查内容，analytics 也能保留更有用的 provider 与入口归因。

## 🐛 修复

### 🛟 恢复与生命周期

- **停止工作会真正停止背后的工作。** 删除对话、删除项目或停止长任务时，会取消对应运行及其子进程，而不是让它们继续消耗时间或额度。

- **重试与会话保持各自边界。** 重试不会继承失败进程的状态；过期 native session 会如实重置；子 agent 结束也不会让父任务过早显示完成。

- **renderer 出问题时，桌面端已有恢复出口。** 连续 renderer 崩溃会进入恢复界面，而不是无限重载；可导出诊断日志跟进问题。Windows 的启动等待、下载/保存与 CLI cache 场景也更稳定。

### 🎨 设计系统、模板与本地化

- **设计系统项目能保留更多你的工作。** 重新生成不会丢失用户文件，workspace 项目重命名可以保留，Swift 色板解析更可靠，缺失预览资源也不会让页面无从继续。

- **资源库更像它本来应该呈现的样子。** 插件分类计数与翻译更可靠，baked preview 更少出现与模板不匹配的情况，deck 预览维持正确比例，WebGL 预览在 resize 时表现更好。

- **界面在更多语言中更完整。** 俄语、泰语与繁体中文等路径覆盖更多更新文案，减少内部标签泄漏到本地化界面。

### 🔒 更安全的边界

- **外部输入会安全地失败。** 品牌提取会拒绝不安全的本地网络目标；损坏的资源流和 proxy 中断不会拖垮 daemon；设计系统解析也更能抵抗异常输入。
