# 靈感來源 NVIDIA 的設計系統

> 類別：媒體與消費性產品
> GPU 運算。黑綠能量，技術力量美學。

## 1. 視覺主題與氛圍

NVIDIA 的網站是一個高對比、以技術為核心的體驗，透過設計克制傳遞出原始運算力量。頁面建立在純黑（`#000000`）與純白（`#ffffff`）的基礎之上，以 NVIDIA 標誌性的綠色（`#76b900`）作為點綴——這個色彩精確到足以成為品牌指紋。這不是自然界的蔥鬱綠意；而是 GPU 渲染光源般的電子感石灰綠，介於黃綠與翠綠之間，在科技圈中能立刻讓人聯想到「NVIDIA」。

客製字型 NVIDIA-EMEA（備援字型為 Arial 與 Helvetica）塑造出簡潔、工業感的排版語調。36px 粗體標題搭配緊密的 1.25 行高，形成密集且具權威感的文字區塊。這套字型缺少矽谷無襯線字型的幾何趣味——它更歐洲化、務實，並帶有工程導向的氣質。內文排在 15–16px，易讀而不奢侈，保持著如同 GPU 記憶體最佳化般的螢幕空間效率感。

NVIDIA 設計與其他深色背景科技網站的差異，在於對綠色強調色的嚴謹運用。`#76b900` 出現在邊框（`2px solid #76b900`）、連結底線（`underline 2px rgb(118, 185, 0)`）以及 CTA 中——但從不作為主要內容區域的背景或大面積色塊。綠色是訊號，不是表面。搭配深沉的陰影系統（`rgba(0, 0, 0, 0.3) 0px 0px 5px`）與極小的圓角（1–2px），整體效果如同精密工程硬體以像素呈現。

**關鍵特徵：**
- NVIDIA 綠色（`#76b900`）作為純粹的強調色——僅用於邊框、底線與互動高亮
- 黑色（`#000000`）主導背景，深色區段搭配白色（`#ffffff`）文字
- NVIDIA-EMEA 客製字型，備援 Arial／Helvetica——工業感、歐洲風、簡潔
- 緊密行高（標題 1.25），形成密集且具權威感的文字區塊
- 極小圓角（1–2px）——全站使用銳利的工程感邊角
- 綠色邊框按鈕（`2px solid #76b900`）作為主要互動模式
- Font Awesome 6 Pro／Sharp 圖示系統，weight 900 呈現銳利圖示
- 多框架架構（PrimeReact、Fluent UI、Element Plus），支援豐富的互動元件

### 原則
- **粗體作為預設語調**：NVIDIA 大量採用 weight 700 用於標題、按鈕、連結與標籤。400 僅保留給內文段落——其餘一律粗體，展現自信與權威感。
- **標題緊密，內文寬鬆**：標題行高固定為 1.25（緊密），而內文文字放鬆至 1.50–1.67。這種對比在內容區塊頂部製造視覺密度，同時保持段落的閱讀舒適度。
- **大寫導覽文字**：連結標籤使用 `text-transform: uppercase` 搭配 weight 700，形成如同硬體規格標籤般的導覽語調。
- **無裝飾性字距**：除緊湊按鈕（0.144px）外，全站字間距均為 normal。字型本身即承載工業感特質，無需額外操作。

## 5. 版面原則

### 間距系統
- 基本單位：8px
- 刻度：1px、2px、3px、4px、5px、6px、7px、8px、9px、10px、11px、12px、13px、15px
- 主要內距值：8px、11px、13px、16px、24px、32px
- 區段間距：48–80px 垂直內距

### 網格與容器
- 最大內容寬度：約 1200px（受限容器）
- 全寬主視覺區段搭配受限文字區塊
- 功能區段：產品卡片使用 2–3 欄網格
- 文章／部落格內容使用單欄
- 文件頁面使用側邊欄排版

### 留白哲學
- **目的性密度**：NVIDIA 使用比一般 SaaS 網站更緊湊的間距，反映技術內容的密集性。留白的存在是為了區隔概念，而非製造奢華的空曠感。
- **區段節奏**：深色區段與白色區段交替出現，透過背景色（而非僅靠間距）區隔內容區塊。
- **卡片密度**：產品卡片緊密排列，間距 16–20px，呈現目錄感而非畫廊感。

### 圓角刻度
- 微型（1px）：行內標籤、極小元素
- 標準（2px）：按鈕、卡片、容器、輸入欄——幾乎所有元素的預設值
- 圓形（50%）：頭像圖片、圓形分頁指示器

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 平面（Level 0） | 無陰影 | 頁面背景、行內文字 |
| 細膩（Level 1） | `rgba(0,0,0,0.3) 0px 0px 5px 0px` | 標準卡片、彈窗 |
| 邊框（Level 1b） | `1px solid #5e5e5e` | 內容分隔線、區段邊框 |
| 綠色強調（Level 2） | `2px solid #76b900` | 啟用元素、CTA、已選取項目 |
| 焦點（無障礙） | `2px solid #000000` 外框 | 鍵盤焦點環 |

**陰影哲學**：NVIDIA 的深度系統極簡且實用。基本上只有一個陰影值——5px 環境模糊、30% 不透明度——僅在卡片與彈窗上節制使用。主要的深度訊號不是陰影，而是_色彩對比_：黑色背景緊鄰白色區段、黑色表面上的綠色邊框。這創造出如同硬體材質的視覺層次，深度源於材質差異，而非模擬光線。

### 裝飾性深度
- 主視覺內容後方的綠色漸層暈染
- 深色到更深色的漸層（黑色到近黑色）用於區段轉場
- 無玻璃擬態或模糊效果——追求清晰而非氛圍感

### 觸控目標
- 按鈕使用 11px 13px 內距，確保舒適的點擊目標
- 導覽連結 14px 大寫搭配適當間距
- 深色背景上的綠色邊框按鈕提供高對比觸控目標
- 手機版：漢堡選單展開為全螢幕覆蓋

### 折疊策略
- 主視覺：36px 標題按比例縮小
- 導覽列：約 1024px 時全水平導覽折疊為漢堡選單
- 產品卡片：3 欄→2 欄→單欄堆疊
- 頁尾：多欄網格折疊為單欄堆疊
- 區段間距：64–80px 在手機版縮減為 32–48px
- 圖片：維持長寬比，縮放至容器寬度

### 圖片行為
- GPU／產品渲染圖在所有尺寸保持高解析度
- 主視覺圖片依視窗比例縮放
- 卡片圖片使用一致的長寬比
- 全出血深色區段維持邊緣到邊緣的處理

## 8. 響應式行為（延伸）

### 字型縮放
- Display 36px 在手機版縮放至約 24px

### 深色／淺色區段策略

- 深色背景：連結為白色，底線為綠色
- 淺色背景：連結為黑色，底線為綠色

### 迭代指南
1. 永遠以 `#76b900` 作為強調色，絕不作為背景填充——它是邊框、底線與高亮的訊號色
2. 按鈕預設為透明搭配綠色邊框——填充背景僅在懸停／啟用狀態出現
3. Weight 700 是所有互動元素與標題的主要語調；400 僅用於內文段落
4. 所有元素圓角統一為 2px——這種銳利的極小圓角是工業美學的核心
5. 深色區段使用白色文字；淺色區段使用黑色文字——綠色強調色在兩者上效果相同
6. 連結懸停一律為 `#3860be`（藍色），無論連結的預設顏色為何
7. 標題行高 1.25，內文行高 1.50–1.67——維持此對比以建立視覺層級
8. 導覽列使用大寫 14px 粗體——這種硬體標籤式的排版語調是品牌語音的一部分
